Databricks Docs Updates

Dashboard enhancements and new capabilities
> ダッシュボードの機能強化と新機能
February 05, 2026
  • Parameterized dashboard tasks: Dashboard tasks in Jobs can now be parameterized, allowing job parameters to dynamically set dashboard filter values at runtime.
  • Workspace-level dashboard themes: Dashboard themes can now be configured at the workspace level. See Manage AI/BI dashhoard workspace themes.
  • Top-aligned text widgets: Text widgets now default to top-aligned text.
  • Refresh-only schedules for individual data permissions: Authors and viewers of dashboards published with individual data permissions can now create refresh-only schedules that trigger cache refreshes without sending notifications, improving dashboard load times. See Subscribe to an existing schedule.
  • Genie spaces enabled by default: Genie is now on by default for published dashboards, allowing viewers to ask questions using the Ask Genie button.
  • Tag search: Tag search is now supported for dashboards, allowing users to find dashboards by their assigned tags.
  • Dashboard authoring agent (Beta): Authors can now automate multi-step dashboard workflows using natural language prompts to create datasets, visualizations, layouts, and filters. See Use the Dashboard Authoring Agent.
>
  • パラメータ化されたダッシュボードタスク: Jobsのダッシュボードタスクはパラメータ化できるようになり、ジョブのパラメーターを使ってダッシュボードのフィルター値を実行時に動的に設定できます。
  • ワークスペースレベルのダッシュボードテーマ: ダッシュボードのテーマをワークスペースレベルで設定できるようになりました。AI/BIダッシュボードのワークスペーステーマの管理を参照してください。
  • 上揃えのテキストウィジェット: テキストウィジェットのデフォルトが上揃えになりました。
  • 個別のデータ権限のための更新専用のスケジュール: 個別のデータ権限で公開されたダッシュボードの作成者と閲覧者は、通知を送らずにキャッシュを更新する更新専用のスケジュールを作成できるようになりました。これによりダッシュボードの読み込み時間が改善されます。既存のスケジュールへの購読を参照してください。
  • デフォルトでGengeスペースが有効: Gengeが公開されたダッシュボードのデフォルトで有効になり、閲覧者がAsk Genieボタンを使って質問できるようになりました。
  • タグ検索: ダッシュボードのタグ検索がサポートされるようになり、ユーザーが割り当てたタグでダッシュボードを見つけられるようになりました。
  • ダッシュボード作成エージェント(ベータ版): 作成者は自然言語プロンプトを使ってデータセット、視覚化、レイアウト、フィルターなどのダッシュボードワークフローを自動化できるようになりました。ダッシュボード作成エージェントの使用を参照してください。

Genie space improvements
> 宇宙開発の改良
February 05, 2026
  • Embedded compute credentials: Compute credentials are now embedded for all users of a Genie space. These credentials are embedded when a Genie space author changes the attached warehouse in the Genie space.
  • Paginated column lists: Column lists in the Configure tab are now paginated for improved usability, especially for rendering tables with many columns.
  • Tag search: Tag search is now supported for Genie spaces, allowing users to find spaces by their assigned tags.
>
  • 組み込みコンピューティング資格情報:Genieスペースのすべてのユーザーに対して、コンピューティング資格情報が組み込まれるようになりました。Genieスペースの作成者が関連付けられたウェアハウスを変更すると、これらの資格情報が組み込まれます。
  • ページネーション付きの列リスト:「設定」タブの列リストは、多くの列を持つテーブルの表示を改善するためにページネーション化されました。
  • タグ検索:Genieスペースに対するタグ検索がサポートされるようになり、ユーザーがそれらに割り当てられたタグでスペースを見つけられるようになりました。
Applying filters, masks, tags, and comments to pipline-created datasets is now GA
> フィルター、マスク、タグ、パイプラインで作成されたデータセットへのコメントの適用がGAになりました。
February 05, 2026

Using CREATE, ALTER, or the Lakeflow UI to modify ETL and ingestion pipelines (Lakeflow Spark Declarative Pipelines and Lakeflow Connect) is now GA. You can modify pipelines to apply row filters, columns masks, table and column tags, column comments, and (for materialized views only) table comments.

See ALTER STREAMING TABLE and ALTER MATERIALIZED VIEW. For general information about using ALTER with Lakeflow Spark Declarative Pipelines, see Use ALTER statements with pipeline datasets.

>

CREATE、ALTER、またはLakeflowのUIを使ってETLおよびインジェストパイプライン(LakeflowのSpark宣言型パイプラインおよびLakeflowConnectパイプライン)を修正することが、ついにガ一般公開されました。パイプラインを修正して、行フィルター、列マスク、テーブルおよび列のタグ、列コメント、ならびに(マテリアライズドビューのみ)テーブルコメントを適用することができます。

ALTER STREAMING TABLEとALTER MATERIALIZED VIEWをご覧ください。LakeflowのSpark宣言型パイプラインでのALTERの使用に関する一般情報については、パイプラインデータセットでALTERステートメントを使うをご参照ください。

Anthropic Claude Opus 4.6 now available as a Databricks-hosted model
> アンソロピック・クロード・オープス4.6がダタブリックスホストモデルで利用可能になりました。
February 05, 2026

Mosaic AI Model Serving now supports Anthropic Claude Opus 4.6 as a Databricks-hosted model.

To access this model, use:

  • Foundation Model APIs pay-per-token
    • Query reasoning models
    • Query vision models
  • Batch inference workloads using AI Functions

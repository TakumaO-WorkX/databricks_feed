Databricks Docs Updates

MLflow metadata is now available in system tables (Public Preview)
> MLflowのメタデータがシステムテーブルで利用可能になりました (パブリックプレビュー)
September 05, 2025

MLflow metadata is now available in system tables. View metadata managed within the MLflow tracking service from the entire workspace in one central location, taking advantage of all the lakehouse tooling Databricks offers, such as building custom AI/BI dashboards, SQL alerts, and large scale data analytic queries.

For information, see MLflow system tables reference.

>

MLflowのメタデータがシステムテーブルで利用可能になりました。Databricksが提供するレイクハウスツーリング、カスタムAI/BIダッシュボード、SQLアラート、大規模なデータ分析クエリを活用して、MLflowトラッキングサービスで管理されるメタデータをワークスペース全体から一元的に表示できます。

詳細については、MLflowシステムテーブルリファレンスを参照してください。

AI/BI dashboard active filter bar and visualization improvements
> AIおよびBIダッシュボードのアクティブフィルターバーおよび可視化の改善
September 04, 2025
  • Active filter bar: A new filter bar shows all active filters on a dashboard. Any filter (global or page-level) not set to All, as well as any cross-filtering, appears in this bar.
  • Minimum and maximum aggregations for date fields: Date fields now support minimum and maximum aggregations.
  • Improved histogram labels: Histograms now display both the start and end of each bin, instead of only the start.
>
  • アクティブなフィルターバー: 新しいフィルターバーには、ダッシュボードのすべてのアクティブなフィルターが表示されます。すべてに設定されていないすべてのフィルター(グローバルまたはページレベル)、およびすべてのクロスフィルタリングが、このバーに表示されます。
  • 日付フィールドの最小値と最大値の集計: 日付フィールドでは、最小値と最大値の集計がサポートされるようになりました。
  • ヒストグラムラベルの改善: ヒストグラムでは、開始ではなく、開始と終了の両方が表示されるようになりました。
New Genie API endpoints and improved benchmarks
> 新しいGenie APIエンドポイントと改善されたベンチマーク
September 04, 2025

  • New API endpoints: The following new endpoints are available for managing and analyzing conversations:

    • List conversation messages
    • Delete conversation message
    • List all conversations

    For more guidance, see Best practices for using the Genie API.

  • Unified permission APIs: Genie now uses the Databricks Permission APIs. See Permissions.

  • Improved Benchmarks evaluation: The evaluation function now classifies results more flexibly. For more information, see Interpret ratings.

>

  • 新しい API エンドポイント: 会話の管理と分析に使用できる以下の新しいエンドポイントが利用可能です:

    • 会話メッセージの一覧取得
    • 会話メッセージの削除
    • すべての会話の一覧取得

    詳細については、Genie API の使用に関するベストプラクティスを参照してください。

  • 統合された権限 API: Genie では、Databricks 権限 API が使用されるようになりました。権限を参照してください。

  • 改善された Benchmarks 評価: 評価関数により、結果がより柔軟に分類されるようになりました。詳細については、評価の解釈を参照してください。

Databricks Assistant Agent Mode: Data Science Agent is in Beta
> データブリックス・アシスタントエージェントモード:データサイエンスエージェントはベータ版にあります。
September 03, 2025

Agent Mode for Databricks Assistant is now in Beta. In Agent Mode, the Assistant can orchestrate multi-step workflows from a single prompt.

The Data Science Agent is custom-built for data science workflows and can build an entire notebook for tasks like EDA, forecasting, and machine learning from scratch. Using your prompt, it can plan a solution, retrieve relevant assets, run code, use cell outputs to improve results, fix errors automatically, and more.

See Use the Data Science Agent.

>

Databricksアシスタントのエージェントモードがベータ版になりました。エージェントモードでは、アシスタントは単一のプロンプトからマルチステップのワークフローを管理できます。

データサイエンスエージェントは、データサイエンスのワークフローのためにカスタムビルドされており、EDA、予測、機械学習などのタスクのために、スクラッチからノートブック全体を作成できます。プロンプトを使用して、ソリューションを計画し、関連するアセットを取得し、コードを実行し、出力を使用して結果を改善し、エラーを自動的に修正することができます。

データサイエンスエージェントの使用を参照してください。

Delta Sharing for tables on default storage will soon be enabled by default (Beta)
> デルタ共有がデフォルトのストレージ上のテーブルでまもなくデフォルトで有効化されます(ベータ版)
September 02, 2025

Delta Sharing providers can share tables backed by default storage to any Delta Sharing recipient (open or Databricks), including recipients using classic compute. This feature is currently in Beta and requires providers to manually turn on the preview. Soon, this will be enabled by default for all users.

See Limitations.

> デルタ共有プロバイダーは、クラシックコンピューティングを使用するレシピエントを含む、任意のデルタ共有レシピエント（オープンまたはデータブリックス）に、デフォルトのストレージでバックアップされたテーブルを共有できます。この機能は現在ベータ版であり、プロバイダーが手動でプレビューをオンにする必要があります。まもなく、すべてのユーザーのためにデフォルトで有効になります。 制限については、「制限」を参照してください。

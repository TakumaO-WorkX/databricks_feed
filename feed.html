Databricks Docs Updates

Dashboard pivot tables, filters, and data export improvements
> ダッシュボードのピボットテーブル、フィルター、データエクスポートの改善
January 08, 2026
  • JSON cell copy-paste improvement: JSON objects in table cells now remain expanded when you drag to copy-paste.
  • Parameterized default catalog and schemas: Parameterize the default catalog and schemas when deploying dashboards using Databricks Asset Bundles. See dashboard.
  • Multi-select filter paste enhancement: Paste comma-separated and newline-separated values into multi-select filters.
  • Pivot table total rows: Pivot tables can now display total rows at the top.
  • Dashboard widget drag handles: Dashboard widgets now have drag handles for resizing and repositioning.
  • Pivoted table Excel downloads: Download dashboard data to Excel in pivoted format rather than flattened. See Download pivot tables to Excel.
  • Timezone display fixes: Fixed a number of timezone-related issues, ensuring that displayed timezones match underlying data timezones instead of displaying UTC.
  • Selective data point labels: Labels can now be shown only for first/last or min/max data points.
>
  • JSONテーブルセルのコピー＆ペースト改善: テーブルセル内のJSONオブジェクトを、コピー＆ペースト時にも展開されたままになるようにしました。
  • パラメータ化されたデフォルトのカタログとスキーマ: Databricks Asset Bundlesを使ってダッシュボードをデプロイする際に、デフォルトのカタログとスキーマをパラメータ化することができます。詳細はダッシュボードをご覧ください。
  • マルチセレクトフィルターのペースト改善: カンマ区切りや改行区切りの値をマルチセレクトフィルターにペーストできるようになりました。
  • ピボットテーブルの合計行: ピボットテーブルに合計行を上部に表示できるようになりました。
  • ダッシュボードウィジェットのドラッグハンドル: ダッシュボードウィジェットにリサイズや位置変更用のドラッグハンドルが追加されました。
  • ピボットされたテーブルのExcel出力: ダッシュボードデータをExcelにピボット形式で出力できるようになりました。詳細はピボットテーブルをExcelにダウンロードをご覧ください。
  • タイムゾーン表示の修正: 基礎データのタイムゾーンではなくUTCが表示されていた問題を修正しました。
  • 選択的なデータポイントラベル: 最初/最後または最小/最大のデータポイントのみラベルを表示できるようになりました。
Genie PDF export, citations, and accuracy improvements
> PDFの書き出し、引用、および正確性の向上
January 08, 2026
  • PDF export for Research Agent (Beta) reports: Export Genie Research Agent reports as PDF files. See Export Research Agent reports.
  • SQL example citations: Genie's analysis steps now cite the SQL examples used. See Review a response.
  • Currency formatting improvements: Numeric columns in table results now use more conservative currency formatting rules to avoid incorrect assumptions.
  • Improved question type handling: Genie now categorizes common question types (top K, percentage, and distribution) more accurately so the right context is applied, improving results.
>
  • Research Agent (ベータ版)の PDF エクスポート: Genie Research Agent レポートを PDF ファイルでエクスポートできます。Research Agent レポートのエクスポートをご覧ください。
  • SQL 例の引用: Genie の分析手順には、使用された SQL 例が引用されるようになりました。レスポンスの確認をご確認ください。
  • 通貨フォーマットの改善: テーブル結果の数値列は、間違った仮定を避けるために、より慎重な通貨フォーマットルールを使用するようになりました。
  • 質問タイプ処理の改善: Genie は、Top K、パーセンテージ、分布などの一般的な質問タイプをより正確に分類するようになり、適切なコンテキストが適用されるため、結果が改善されます。
Genie value sampling features renamed
> ジーニーの値サンプリング機能が名称変更されました。
January 07, 2026

The value sampling features in Genie will soon be renamed to better describe their function:

  • Value sampling will be renamed to prompt matching
  • Example values will be renamed to format assistance
  • Value dictionaries will be renamed to entity matching
>

Genieの値サンプリング機能は、その機能をより適切に表現するよう近日中に名称変更されます:

  • 値サンプリングは、プロンプトマッチングに名称変更されます
  • サンプル値は、フォーマットアシストに名称変更されます
  • 値辞書は、エンティティマッチングに名称変更されます
Change default workspace access to Consumer access (Public Preview)
> コンシューマーアクセス（パブリックプレビュー）にデフォルトのワークスペースアクセスを変更する
January 06, 2026

Workspace admins can configure new users to receive Consumer access by default using group cloning. Consumer access provides access to Databricks One, a view-only experience for business users who need to view and interact with dashboards, Genie spaces, and apps without creating workspace objects. The process creates a new group for existing users and updates the users group to grant only Consumer entitlement to new users by default. This preserves existing users' authoring privileges while streamlining consumer onboarding at scale. See Change default workspace access to Consumer access.

> ワークスペースの管理者は、グループのクローン作成を使用して、新しいユーザーにデフォルトでコンシューマーアクセスを設定できます。コンシューマーアクセスは、ダッシュボード、Genieスペース、およびアプリケーションを表示および操作する必要がある業務ユーザーのためのビューのみの体験を提供します。このプロセスでは、既存のユーザーの新しいグループが作成され、「ユーザー」グループが更新されてデフォルトで新しいユーザーにコンシューマー権限のみが付与されます。これにより、既存ユーザーの編集権限が保持されつつ、大規模なコンシューマー登録が効率化されます。ワークスペースアクセスをコンシューマーアクセスに変更するをご覧ください。
New slicing logic for jobs timeline tables
> 新しい仕事タイムラインテーブルのスライス論理
January 06, 2026

Starting January 19, 2026, the jobs timeline tables use a new clock-hour-aligned slicing logic. Time slices now align to standard clock hour boundaries (5:00-6:00 PM, 6:00-7:00 PM, and so on) instead of one-hour intervals based on the run's start time. New rows will use the new slicing logic, while existing rows remain unchanged.

See Clock-hour-aligned slicing logic.

> 2026年1月19日から、ジョブのタイムラインテーブルは新しい時計時間単位の分割ロジックを使用します。時間スライスは、実行開始時刻に基づくではなく、標準の時計時間の境界(午後5:00-6:00、午後6:00-7:00など)に合わせて設定されます。新しい行は新しい分割ロジックを使用しますが、既存の行は変更されません。 時計時間単位の分割ロジックを参照してください。

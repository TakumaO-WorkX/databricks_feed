Databricks Docs Updates

Dashboard enhancements and improvements
> ダッシュボードの強化と改善
March 05, 2026
  • Embedded dashboard kebab menu: The Copy link to widget option is no longer shown in the kebab menu for embedded dashboards.
  • Dashboard import warehouse ID fix: Importing a dashboard without a specified warehouse ID no longer clears the previously selected warehouse.
  • Legacy dashboards retirement: Legacy dashboards (previously referred to as Databricks SQL dashboards) are no longer available.
  • Bar and line chart x-axis spacing fix: The x-axis spacing of temporal data labels is now consistent between bar and line charts.
  • Pivot table drill through: You can now drill through data in pivot tables.
  • Point map recentering on filter changes: Point maps automatically recenter to show relevant data when a filter is applied.
  • AI chart suggestion fix: AI generated chart suggestions no longer disappear in some scenarios.
  • Waterfall chart faceting: You can now apply faceting (also known as trellis or small multiple charts) to waterfall charts.
  • Unity Catalog metric views support in Excel and Google Sheets: You can now import Unity Catalog metric views as pivot tables using the Databricks Excel Add-in and the Databricks Connector for Google Sheets.
>
  • 埋め込みダッシュボードのケバブメニュー: ウィジェットのリンクをコピーオプションは、埋め込みダッシュボードのケバブメニューに表示されなくなりました。
  • ダッシュボードのインポートウェアハウスIDの修正: 指定されたウェアハウスIDなしでダッシュボードをインポートすると、以前に選択されていたウェアハウスがクリアされなくなりました。
  • レガシーダッシュボードの廃止: レガシーダッシュボード(以前はDatabricks SQLダッシュボードと呼ばれていた)は利用できなくなりました。
  • 棒グラフと折れ線グラフのx軸スペーシングの修正: 時系列データラベルのx軸スペーシングが、棒グラフと折れ線グラフの間で一致するようになりました。
  • ピボットテーブルのドリルスルー: ピボットテーブルのデータにドリルスルーできるようになりました。
  • フィルター変更時のポイントマップの再中心化: フィルターを適用すると、ポイントマップが関連するデータを表示するように自動的に再中心化されます。
  • AIチャート提案の修正: 一部のシナリオでAI生成のチャート提案が消えなくなりました。
  • ウォーターフォールチャートのファセティング: ウォーターフォールチャートにファセティング(トレリスまたは小さな複数のチャートとしても知られる)を適用できるようになりました。
  • ExcelとGoogle Sheetsでのユニティカタログメトリックビューのサポート: Databricks ExcelアドインとDatabricks Connector for Google Sheetsを使って、ユニティカタログメトリックビューをピボットテーブルとしてインポートできるようになりました。
OpenAI GPT-5.4 now available as a Databricks-hosted model
> オープンAI GPT-5.4がDatabricksのホストモデルとして利用可能になりました。
March 05, 2026

Mosaic AI Model Serving now supports OpenAI GPT-5.4 as a Databricks-hosted model. You can access this model using Foundation Model APIs pay-per-token.

Customers are responsible for ensuring their compliance with the terms of OpenAI's Acceptable Use Policy.

> モザイクAIモデルサービングにはOpenAI GPT-5.4がデータブリックスホストのモデルとしてサポートされています。この モデルにはFoundation Model APIs pay-per-tokenを使ってアクセスできます。 お客様は、OpenAIの使用許可ポリシーの条件を遵守する責任があります。
Tree view of the workspace browser side panel (Public Preview)
> ワークスペースブラウザのサイドパネル内のツリービュー（パブリックプレビュー）
March 05, 2026

The workspace browser side panel now includes a tree view that provides a flexible, tree-based view of your workspace folder hierarchy. The tree view allows you to expand descendant folders, change the root of the tree, and navigate your workspace while keeping the tree steady. This improves the multi-asset authoring experience by providing better control over workspace navigation.

See Tree view of the workspace browser side panel (Public Preview).

> ワークスペース・ブラウザ側面パネルには現在、ワークスペース・フォルダ階層を柔軟なツリー形式で表示するツリービューが含まれています。ツリービューでは、子フォルダを展開したり、ツリーのルートを変更したり、ツリーを安定させたままでワークスペースを移動したりすることができます。これにより、ワークスペースの移動操作がより容易になり、マルチアセットの制作体験が向上します。 「ワークスペース・ブラウザ側面パネルのツリービュー (パブリックプレビュー)」をご覧ください。
Lakebase Autoscaling now available with compliance security profile
> レイクベースの自動スケーリングがコンプライアンスセキュリティプロファイルで利用可能になりました。
March 04, 2026

Lakebase Autoscaling is now available for workspaces with the compliance security profile when the compliance standard is set to HIPAA, C5, TISAX, or None.

See Lakebase Autoscaling limitations.

>

Lakebaseオートスケーリングは、コンプライアンス標準がHIPAA、C5、TISAXまたはNoneに設定されている場合、コンプライアンスセキュリティプロファイルを持つワークスペースで利用可能になりました。

Lakebaseオートスケーリングの制限については、こちらをご確認ください。

Public URLs for ODBC driver downloads will be disabled on March 27
> https://download.microsoft.com/download/8/E/F/8EF6F5B1-8546-4F8D-BDFB-6A3FD18CC1C3/odbc_driver_for_sql_server.exe
March 03, 2026

On March 27, 2026, public S3 links for downloading the Simba Spark ODBC Driver will be disabled. After this date, scripts or processes that use direct public links to download the driver will no longer function. The driver will be available exclusively through the ODBC archive page, which requires sign-in.

To continue using automated downloads, migrate to the Databricks ODBC driver before March 27, 2026. See Migrate from Simba Spark ODBC Driver to Databricks ODBC Driver.

> 2026年3月27日以降、Simba Spark ODBCドライバーのダウンロード用の公開S3リンクが無効になります。この日以降、直接のパブリックリンクを使ってドライバーをダウンロードするスクリプトやプロセスは機能しなくなります。ドライバーはODBC アーカイブページでのみ利用可能で、そこではログインが必要です。 自動ダウンロードを続けるには、2026年3月27日までにDatabricks ODBCドライバーに移行してください。Simba Spark ODBCドライバーからDatabricks ODBCドライバーへの移行を参照してください。

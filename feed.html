Databricks Docs Updates

Create backfill job runs
> ジョブ実行の埋め戻し作成
October 14, 2025

Job backfills allow you to trigger job runs to backfill data from the past. This is useful for loading older data, or repairing data when there are failures in processing. For more details, see Backfill jobs.

> ジョブのバックフィルは、過去からデータを取り込むためのジョブを実行するのに使えます。これは、古いデータをロードしたり、処理の失敗を修復するのに役立ちます。詳細については、バックフィルジョブをご覧ください。
Data Classification (Public Preview)
> データ分類 (パブリックプレビュー)
October 13, 2025

Databricks Data Classification is now in Public Preview and supports all catalog types, consolidates all classification results into a single system table, and a new UI to review and auto-tag classifications. See Data Classification.

> データブリックスのデータ分類がパブリックプレビューに登場し、すべてのカタログタイプをサポートし、分類結果をすべて単一のシステムテーブルに統合し、分類を確認および自動タグ付けするための新しいUIを提供しています。「データ分類」をご覧ください。
Multimodal support is now available
> マルチモーダルサポートが今後利用可能となります。
October 13, 2025

Mosaic AI Model Serving now supports multimodal inputs for the Databricks hosted foundation models. See Query vision models.

This multimodal support is available using the following functionalities:

  • Foundation Model APIs pay-per-token.
  • Foundation Model APIs provisioned throughput.
  • AI Functions. Both real-time inference and batch inference workloads.
>

Databricks社ホスト型基盤モデルに対するマルチモーダル入力をMosaic AIモデルサーバーがサポートするようになりました。Query vision modelsをご覧ください。

このマルチモーダルサポートは以下の機能を使って利用できます:

  • 基盤モデルAPI トークン従量課金。
  • 基盤モデルAPI 処理能力確保。
  • AIファンクション。リアルタイム推論およびバッチ推論ワークロード。

AI/BI dashboard table and text widget improvements
> AIとBIのダッシュボードのテーブルとテキストウィジェットの改善
October 09, 2025
  • Copy link URLs from tables: You can now copy link URLs directly from table visualizations.
  • Improved pivot row headers: Row header field names in pivot tables no longer overflow and now scroll together with row values.
  • Improved handling of < character in text widgets: Text widgets now correctly handle the < character to prevent unexpected behavior.
  • Improved precision for custom calculations: Aggregate functions now support the DISTINCT keyword and the FILTER(WHERE ...) clause. Use them to count unique entities and perform conditional aggregation, replacing complex CASE statements. See Aggregate functions.
  • Additional predicate support for custom calculations: Custom calculations now support IN, BETWEEN, and pattern matching predicates (LIKE, ILIKE, RLIKE) for more flexible filtering and comparison operations. See What are custom calculations?.
>
  • テーブルからリンクURLをコピーする: テーブルの視覚化からリンクURLを直接コピーできるようになりました。
  • ピボットテーブルのヘッダーが改善されました: ピボットテーブルの行ヘッダーフィールド名が溢れることなく、行の値とともにスクロールするようになりました。
  • テキストウィジェットの<文字の処理が改善されました: テキストウィジェットで<文字が正しく処理されるようになり、予期せぬ動作が防げるようになりました。
  • カスタム計算の精度が改善されました: 集計関数でDISTINCTキーワードとFILTER(WHERE ...)句がサポートされるようになりました。これを使ってユニークなエンティティをカウントしたり、条件付き集計を行ったりすることができ、複雑なCASEステートメントを置き換えることができます。集計関数を参照してください。
  • カスタム計算の述語サポートが追加されました: カスタム計算でIN、BETWEEN、パターンマッチング述語(LIKE、ILIKE、RLIKE)がサポートされるようになり、より柔軟なフィルタリングと比較処理が可能になりました。カスタム計算とは何ですか?を参照してください。

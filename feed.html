Databricks Docs Updates

Select tables and create pivot tables in Google Sheets
> Google Sheetsでテーブルを選択し、ピボットテーブルを作成する
February 03, 2026

You can now directly select Databricks tables from the catalog explorer and import data as pivot tables in Google Sheets using the Databricks Connector. See Connect to Databricks from Google Sheets.

>

データブリックステーブルをカタログエクスプローラから直接選択し、データブリックスコネクタを使ってGoogleスプレッドシートでピボットテーブルとしてデータをインポートできるようになりました。Googleスプレッドシートからデータブリックスに接続するをご覧ください。

Regional model hosting for Genie in Japan and Korea
> 地域モデルホスティング for Genie in 日本 and 韓国
February 02, 2026

For workspaces in ap-northeast-1 (Tokyo) and ap-northeast-2 (Seoul), Genie now uses models hosted in the same region. Users in these regions no longer require cross-Geo processing to use Genie.

See Availability of Designated Services in each Geo for more information on which features require cross-geo processing.

>

ap-northeast-1(東京)およびap-northeast-2(ソウル)のワークスペースの場合、Genieは同じリージョンでホストされているモデルを使用するようになりました。これらのリージョンのユーザーは、Genieを使用するためにもはやクロスGeo処理を必要としません。

クロスGeo処理を必要とする機能については、各Geoでの指定サービスの可用性を参照してください。

Tag Databricks Apps (Public Preview)
> タグ Databricks Apps (パブリックプレビュー)
February 02, 2026

You can now apply tags to Databricks apps to organize and categorize them. See Apply tags to Databricks apps. Search is not supported using Databricks apps tags.

>

Databricksのアプリにタグを適用して、それらを整理およびカテゴリ化できるようになりました。Databricksのアプリにタグを適用するをご確認ください。Databricksのアプリタグを使用した検索はサポートされていません。

Zendesk Support connector (Beta)
> Zendesk サポートコネクタ (ベータ版)
February 02, 2026

The Zendesk Support connector allows you to ingest ticket data, help center content, and community forum data from Zendesk Support. See Zendesk Support connector overview.

> Zendeskサポートコネクタはチケットデータ、ヘルプセンターのコンテンツ、およびZendeskサポートからのコミュニティフォーラムデータを取り込むことができます。Zendeskサポートコネクタの概要を参照してください。
Data Quality Monitoring Anomaly Detection public preview
> データ品質モニタリング異常検知パブリックプレビュー
February 02, 2026

Databricks Data Quality Monitoring Anomaly Detection is now in Public Preview. The feature is enabled at the schema level and learns from historical data patterns to detect data quality anomalies. The health of all monitored tables is consolidated into a single system table and a new UI. See Anomaly detection.

> Databricks のデータ品質モニタリング異常検出機能が一般公開プレビューになりました。この機能はスキーマレベルで有効化され、過去のデータパターンを学習して、データ品質の異常を検出します。監視対象のすべてのテーブルの健全性は、単一のシステムテーブルと新しい UI に集約されます。異常検出の詳細をご確認ください。

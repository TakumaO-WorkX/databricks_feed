Changes to open Delta Sharing recipient tokens
> [翻訳エラー: Error code: 400 - {'type': 'error', 'error': {'type': 'invalid_request_error', 'message': 'Your credit balance is too low to access the Anthropic API. Please go to Plans & Billing to upgrade or purchase credits.'}, 'request_id': 'req_011CZTNw4UCv1X3ZPygVv3m9'}]
March 19, 2026

Delta Sharing for open recipients will transition to a new recipient-specific URL format. The transition date has been updated and is now July 1, 2026. New tokens created on or after July 1, 2026 will automatically use the new URL format. This change improves network security and lets recipients configure recipient-specific network policies and firewall rules.

For AWS GovCloud and AWS GovCloud DoD, the transition will be announced later.

The new URLs include the recipient ID in the domain:

https://<recipient-id>.delta-sharing.us-west-2.cloud.databricks.com/api/2.0/delta-sharing/metastores/<metastore-id>

For reference, URLs created before this change do not contain the recipient ID.

https://oregon.cloud.databricks.com/api/2.0/delta-sharing/metastores/<metastore-id>

The old URLs will continue to work for a period of time. The specific duration depends on the recipient type and the token creation date. Data providers should transition to the new URL format before the old URL format becomes invalid.

OIDC Federation sharing:

Data providers need to verify their recipients are using the new URL format before July 1, 2027. Starting July 1, 2026, providers can find the new URL in the Delta Sharing UI. After July 1, 2027, the old URL format will not be valid.

Bearer token sharing:

    • Token creation date
    • URL format
    • Token expiration date
    • Recommended action
    • Before July 1, 2026
    • Old format
    • One year from creation date or December 8, 2026, whichever date is further in the future
    • Data providers need to rotate tokens before expiration to migrate to the new URL format. To provide recipients time to migrate, configure a downtime window by setting an expiration date for the current token during rotation. Both old and new URL formats are supported during this period.
    • After July 1, 2026
    • New format
    • Per your configuration, up to one year from creation date.

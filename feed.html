Databricks Docs Updates

Deploy Databricks apps from Git repositories (Beta)
> GitリポジトリからのDatabricksアプリのデプロイ(ベータ)
February 06, 2026

You can now deploy Databricks apps directly from Git repositories without uploading files to the workspace. Configure a repository for your app and deploy from any branch, tag, or commit. See Deploy from a Git repository.

>

データブリックスのアプリケーションをワークスペースにファイルをアップロードせずに、Git リポジトリから直接デプロイできるようになりました。アプリケーション用のリポジトリを構成し、任意のブランチ、タグ、またはコミットからデプロイできます。詳細は Git リポジトリからデプロイをご覧ください。

Query tags for SQL warehouses (Public Preview)
> クエリタグ for SQL ウェアハウス (パブリックプレビュー)
February 06, 2026

You can now apply custom key-value tags to SQL workloads on Databricks SQL warehouses for grouping, filtering, and cost attribution. Query tags appear in the system.query.history table and on the Query History page of the Databricks UI, allowing you to attribute warehouse costs by business context and identify sources of long-running queries. Tags can be set using session configuration parameters, the SET QUERY_TAGS SQL statement, or through connectors including dbt, Power BI, Tableau, Python, Node.js, Go, JDBC, and ODBC. See Query tags.

> データブリックスSQLウェアハウスの SQLワークロードに対してカスタムキーバリューのタグを適用し、グループ化、フィルタリング、コスト属性付けが可能になりました。クエリタグはsystem.query.historyテーブルと、データブリックスのUIのクエリ履歴ページに表示されるため、ビジネスコンテキストに基づいてウェアハウスコストを割り当てたり、長時間実行クエリの発生源を特定できます。タグは、セッション構成パラメータ、SET QUERY_TAGS SQL文、または dbt、Power BI、Tableau、Python、Node.js、Go、JDBC、ODBCなどのコネクタを使って設定できます。詳細はクエリタグをご覧ください。
Dashboard enhancements and new capabilities
> ダッシュボードの機能強化と新機能
February 05, 2026
  • Parameterized dashboard tasks: Dashboard tasks in Jobs can now be parameterized, allowing job parameters to dynamically set dashboard filter values at runtime.
  • Workspace-level dashboard themes: Dashboard themes can now be configured at the workspace level. See Manage AI/BI dashhoard workspace themes.
  • Top-aligned text widgets: Text widgets now default to top-aligned text.
  • Refresh-only schedules for individual data permissions: Authors and viewers of dashboards published with individual data permissions can now create refresh-only schedules that trigger cache refreshes without sending notifications, improving dashboard load times. See Subscribe to an existing schedule.
  • Genie spaces enabled by default: Genie is now on by default for published dashboards, allowing viewers to ask questions using the Ask Genie button.
  • Tag search: Tag search is now supported for dashboards, allowing users to find dashboards by their assigned tags.
  • Dashboard authoring agent (Beta): Authors can now automate multi-step dashboard workflows using natural language prompts to create datasets, visualizations, layouts, and filters. See Use the Dashboard Authoring Agent.
>
  • パラメータ化されたダッシュボードタスク: ジョブのダッシュボードタスクは、今後パラメータ化されるようになり、ジョブパラメータを使ってダッシュボードフィルターの値をランタイムで動的に設定できるようになります。
  • ワークスペースレベルのダッシュボードテーマ: ダッシュボードのテーマは、ワークスペースレベルで設定できるようになりました。AI/BIダッシュボードワークスペースのテーマ管理をご覧ください。
  • トップ位置合わせのテキストウィジェット: テキストウィジェットはデフォルトでトップ位置合わせのテキストになります。
  • 個別データ権限を持つリフレッシュのみのスケジュール: 個別データ権限で公開されたダッシュボードの作成者と閲覧者は、通知を送信せずにキャッシュリフレッシュをトリガーするリフレッシュのみのスケジュールを作成できるようになりました。ダッシュボードの読み込み時間が改善されます。既存のスケジュールに登録するをご覧ください。
  • デフォルトで有効化されたGenie spaces: Genieは今後公開されたダッシュボードにデフォルトで有効になり、閲覧者はAsk Genieボタンを使って質問できるようになります。
  • タグ検索: ダッシュボードのタグ検索がサポートされるようになり、割り当てられたタグでダッシュボードを検索できるようになりました。
  • ダッシュボードの作成エージェント(ベータ版): 作成者は、自然言語のプロンプトを使ってデータセット、可視化、レイアウト、フィルターなどのマルチステップのダッシュボードワークフローを自動化できるようになりました。ダッシュボードの作成エージェントを使うをご覧ください。

Genie space improvements
> ジーニースペースの改善
February 05, 2026
  • Embedded compute credentials: Compute credentials are now embedded for all users of a Genie space. These credentials are embedded when a Genie space author changes the attached warehouse in the Genie space.
  • Paginated column lists: Column lists in the Configure tab are now paginated for improved usability, especially for rendering tables with many columns.
  • Tag search: Tag search is now supported for Genie spaces, allowing users to find spaces by their assigned tags.
>
  • 組み込み計算資格情報: ジーニースペースのすべてのユーザーに計算資格情報が組み込まれるようになりました。これらの資格情報は、ジーニースペースの作成者が関連付けられたウェアハウスを変更した際に組み込まれます。
  • ページ分割列一覧: [Configure]タブの列一覧は、特に多くの列のテーブルをレンダリングするためのユーザビリティを改善するためにページ分割されるようになりました。
  • タグ検索: ジーニースペースでタグ検索がサポートされるようになり、割り当てられたタグによってスペースを見つけることができるようになりました。

Upcoming behavioral change: VOID columns included in Delta table reads
> [翻訳エラー: Error code: 400 - {'type': 'error', 'error': {'type': 'invalid_request_error', 'message': 'Your credit balance is too low to access the Anthropic API. Please go to Plans & Billing to upgrade or purchase credits.'}, 'request_id': 'req_011Ca5P2MRCsx5VpUjwR2YY4'}]
April 13, 2026

In mid-June 2026, Delta Lake will fully support VOID columns. Previously, VOID columns were silently skipped by path-based DataFrame reads (for example, spark.read.format("delta").load(path)) and time travel queries. After this change, these queries include VOID columns in the output.

Queries that depend on column count or position, such as INSERT INTO ... SELECT *, may fail or produce incorrect results after this change. Review any queries that read from Delta Lake tables with VOID columns to ensure they handle the additional columns correctly.

See VOID type.

Column-level personal data governance tags from SAP Business Data Cloud (BDC) now sync into Unity Catalog, including fieldSemantics, isPotentiallyPersonal, and isPotentiallySensitive. Use these tags in attribute-based access control (ABAC) policies to govern access to sensitive columns. See SAP BDC semantic metadata.

Serverless GPU compute now supports H100 accelerators (Beta)
> Serverless GPUコンピューティングが今H100アクセラレータ(ベータ版)に対応しました。
September 05, 2025

Serverless GPU compute now supports on-demand H100 accelerators for single-node workflows and jobs. This feature is in Beta.

See Serverless GPU compute.

> サーバーレスGPUコンピューティングは、単一ノードのワークフローおよびジョブに対して、オンデマンドのH100アクセラレーターをサポートするようになりました。このフィーチャーはベータ版です。 サーバーレスGPUコンピューティングをご覧ください。
Databricks Online Feature Stores (Public Preview)
> データブリックスオンラインフィーチャーストア(パブリックプレビュー)
September 05, 2025

Databricks Online Feature Stores, powered by Lakebase, provide highly-scalable low-latency access to feature data while maintaining consistency with your offline feature tables. Native integrations with Unity Catalog, MLflow, and Mosaic AI Model Serving help you productionize your model endpoints, agents, and rule engines, so they can automatically and securely access features from Online Feature Stores while maintaining high performance.

See Databricks Online Feature Stores.

>

Lakebaseによって駆動されるDatabricksオンラインフィーチャーストアは、オフラインのフィーチャーテーブルとの整合性を維持しながら、高スケーラビリティとローレイテンシーのフィーチャーデータアクセスを提供します。Unity Catalog、MLflow、Mosaic AIモデルサービングとのネイティブ統合により、モデルエンドポイント、エージェント、ルールエンジンを本番稼働させることができ、それらがオンラインフィーチャーストアからの高パフォーマンスでのデータアクセスを自動的かつ安全に行えるようになります。

Databricksオンラインフィーチャーストアを参照してください。

MLflow metadata is now available in system tables (Public Preview)
> MLflowのメタデータがシステムテーブルで利用可能になりました(パブリックプレビュー)
September 05, 2025

MLflow metadata is now available in system tables. View metadata managed within the MLflow tracking service from the entire workspace in one central location, taking advantage of all the lakehouse tooling Databricks offers, such as building custom AI/BI dashboards, SQL alerts, and large scale data analytic queries.

For information, see MLflow system tables reference.

> MLフローのメタデータは、システムテーブルで利用可能になりました。Databricksが提供するレイクハウスツーリング、カスタムAI/BIダッシュボード、SQLアラート、大規模なデータ分析クエリなどを活用して、MLフロートラッキングサービスで管理されているメタデータを、ワークスペース全体の中央の場所から表示できます。 詳細については、MLフローのシステムテーブルのリファレンスをご覧ください。
AI/BI dashboard active filter bar and visualization improvements
> AIおよびBIのダッシュボードのアクティブなフィルターバーと視覚化の改善
September 04, 2025
  • Active filter bar: A new filter bar shows all active filters on a dashboard. Any filter (global or page-level) not set to All, as well as any cross-filtering, appears in this bar.
  • Minimum and maximum aggregations for date fields: Date fields now support minimum and maximum aggregations.
  • Improved histogram labels: Histograms now display both the start and end of each bin, instead of only the start.
>
  • アクティブフィルターバー: 新しいフィルターバーでは、ダッシュボードのすべてのアクティブフィルターが表示されます。すべてに設定されていないグローバルまたはページレベルのフィルター、およびクロスフィルタリングがこのバーに表示されます。
  • 日付フィールドの最小値と最大値の集計: 日付フィールドでは、最小値と最大値の集計がサポートされるようになりました。
  • ヒストグラムラベルの改善: ヒストグラムでは、各ビンの開始のみでなく、終了も表示されるようになりました。
New Genie API endpoints and improved benchmarks
> 新しいGenie APIのエンドポイントと改善されたベンチマーク
September 04, 2025

  • New API endpoints: The following new endpoints are available for managing and analyzing conversations:

    • List conversation messages
    • Delete conversation message
    • List all conversations

    For more guidance, see Best practices for using the Genie API.

  • Unified permission APIs: Genie now uses the Databricks Permission APIs. See Permissions.

  • Improved Benchmarks evaluation: The evaluation function now classifies results more flexibly. For more information, see Interpret ratings.

>

  • 新しいAPI エンドポイント: 会話の管理と分析のために、以下の新しいエンドポイントが利用できるようになりました:

    • 会話メッセージの一覧取得
    • 会話メッセージの削除
    • すべての会話の一覧取得

    詳しくは、Genieのアプリケーションプログラミングインターフェース(API)の利用ガイドラインをご覧ください。

  • 統一化されたパーミッションAPI: GenieはDatabricksのパーミッションAPIを使用するようになりました。パーミッションを参照してください。

  • 改善されたベンチマーク評価: 評価関数が結果を柔軟に分類するようになりました。詳細については、評価結果の解釈をご覧ください。

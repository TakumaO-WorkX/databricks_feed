Lakeflow Spark Declarative Pipelines stream progress metrics are generally available
> レイクフロースパーク宣言型パイプラインのストリームプログレスメトリクスが一般に利用可能となりました。

Lakeflow Spark Declarative Pipelines support for querying the event log for metrics about the progress of a stream is now generally available. See Monitor pipeline streaming metrics.

>

ラクフロースパーク宣言パイプラインでは、ストリームの進捗に関するメトリクスを取得するためのイベントログのクエリがサポートされるようになりました。詳細はパイプラインのストリーミングメトリクスの監視をご確認ください。

Databricks Assistant Agent Mode preview
> データブリックス アシスタント エージェントモード プレビュー
November 21, 2025

The Databricks Assistant Agent Mode preview will soon be enabled by default for most customers.

  • The agent can automate multiple steps. From a single prompt, it can retrieve relevant assets, generate and run code, fix errors automatically, and visualize results. It adds the ability to sample data and cell outputs to provide better results.
  • The Assistant in Agent Mode will choose between Azure OpenAI or Anthropic on Databricks (uses endpoints hosted by Databricks Inc. in AWS within the Databricks security perimeter), and is only available when the partner-powered AI features setting is enabled.
  • Admins can disable the preview if needed until the feature reaches General Availability.

See Use the Data Science Agent, the blog post, and Partner-powered AI features.

>

Databricks Assistantエージェントモードプレビューが、ほとんどのお客様のデフォルトで間もなく有効になります。

  • エージェントは複数のステップを自動化できます。単一のプロンプトから、関連するアセットを取得し、コードを生成して実行し、エラーを自動的に修正し、結果を視覚化できます。データのサンプリングとセル出力の機能を追加して、より良い結果を提供します。
  • エージェントモードのアシスタントは、Databricksでホストされているエンドポイントを使用して、Azure OpenAIまたはAnthropic AIを選択します。パートナー提供のAI機能の設定が有効になっている場合にのみ利用できます。
  • 管理者は、この機能が一般提供される前に、必要に応じてプレビューを無効にできます。

データサイエンスエージェントの使用、ブログ記事、およびパートナー提供のAI機能をご覧ください。

Configure full refresh behavior for managed database connectors
> データベースコネクターの完全更新動作を構成する
November 21, 2025

You can now schedule when full refresh snapshots occur and enable automatic full refresh to recover from unsupported schema changes. This applies to managed database connectors like SQL Server in Lakeflow Connect. See Configure full refresh behavior for database connectors.

> 今、完全リフレッシュのスナップショットが発生するタイミングをスケジュールできるようになりました。また、サポートされていないスキーマの変更から回復するために、自動的な完全リフレッシュを有効にできるようになりました。これは、Lakeflow ConnectのSQL Serverなどの管理されたデータベースコネクタに適用されます。「データベースコネクタの完全リフレッシュの動作を設定する」をご覧ください。
Automate classic workspace creation with AWS IAM temporary delegation
> AWSのIAM一時的な委任を使ってクラシックワークスペースの作成を自動化する
November 21, 2025

You can now automate the creation of classic workspaces using AWS IAM temporary delegation, which grants Databricks temporary access to provision the required IAM roles, S3 bucket, access policies, and customer-managed VPC in your AWS account. This workspace creation flow replaces the AWS Quickstart template.

See Create a classic workspace.

> AWS IATMの一時的な委任を使用して、クラシック・ワークスペースの作成を自動化できるようになりました。これにより、Databricksに、必要なIAMロール、S3バケット、アクセスポリシー、およびカスタマー管理VPCをプロビジョニングするための一時的なアクセス権限が付与されます。このワークスペース作成フローは、AWS Quickstartテンプレートに置き換わるものです。 クラシック・ワークスペースの作成をご覧ください。
Real-time collaboration in notebook cells, files, and SQL editor
> リアルタイムのノートブックセル、ファイル、SQLエディタでのコラボレーション
November 21, 2025

Real-time collaboration is now available in notebook cells, files, and the new SQL editor. Multiple users can edit the same cell simultaneously and can view each other's edits. For more information, see Collaborate using Databricks notebooks.

> リアルタイムのコラボレーションがノートブックセル、ファイル、および新しいSQLエディターで利用可能になりました。複数のユーザーが同じセルを同時に編集し、お互いの編集を表示できます。詳細については、「Databricksノートブックを使用してコラボレーションする」を参照してください。

Databricks Docs Updates

Default warehouse setting is now available in Beta
> デフォルトの倉庫設定がベータ版で利用可能になりました。
August 28, 2025

Set a default warehouse that will be automatically selected in the compute selector across the SQL editor, AI/BI dashboards, AI/BI Genie, Alerts, and Catalog Explorer. Individual users can override this setting by selecting a different warehouse before running a query. They can also define their own user-level default warehouse to apply across their sessions. See Set a default SQL warehouse for the workspace and Set a user-level default warehouse.

> SQLエディター、AI/BI ダッシュボード、AI/BI ジーニー、アラート、およびCatalog Explorerにわたる計算セレクター内で自動的に選択されるデフォルトのウェアハウスを設定します。個々のユーザーは、クエリの実行前に異なるウェアハウスを選択することで、この設定を上書きできます。また、自分のセッションに適用するユーザーレベルのデフォルトウェアハウスを定義することもできます。「ワークスペースのデフォルトのSQL ウェアハウスを設定する」と「ユーザーレベルのデフォルトのウェアハウスを設定する」を参照してください。
Track and navigate notebook runs with the new cell execution minimap
> セル実行ミニマップを搭載した新しいノートブック走行を追跡し、ナビゲートしてください。
August 28, 2025

Use the cell execution minimap to track your notebook’s progress at a glance. The minimap appears in the right margin and shows each cell’s execution state (skipped, queued, running, success, or error). Hover to see cell details, or click to jump directly to a cell.

For information about using the cell execution minimap, see Navigate the Databricks notebook and file editor.

> ノートブックの進捗状況を一目で確認できるセル実行ミニマップを使用してください。ミニマップは右端に表示され、各セルの実行状態(スキップ、待機、実行中、成功、エラー)を示します。セルの詳細を表示するにはマウスを重ねてください。クリックすると直接そのセルに移動できます。 セル実行ミニマップの使用方法については、「Databricksノートブックとファイルエディターの操作」を参照してください。
Admins can now manage a workspace's serverless base environments (Public Preview)
> 管理者は、ワークスペースのサーバーレスベース環境を管理できるようになりました(パブリックプレビュー)
August 28, 2025

Base environments are custom environment specifications for serverless notebooks that define a serverless environment version and a set of dependencies.

Workspace admins can now create and manage the base environments available in their workspace. They can also set a default base environment for all new serverless notebooks. This allows workspace users to quickly start working in a consistent, cached environment.

See Manage serverless base environments.

> サーバーレスノートブックのカスタム環境仕様であるベースの環境は、サーバーレス環境のバージョンと依存関係のセットを定義します。 ワークスペースの管理者は、ワークスペースで使用可能なベース環境の作成と管理ができます。また、すべての新しいサーバーレスノートブックのデフォルトのベース環境を設定することもできます。これにより、ワークスペースのユーザーは、一貫して、キャッシュされた環境で素早く作業を開始できます。 「サーバーレスベース環境の管理」をご覧ください。
Lakebase synced tables supports syncing Apache Iceberg and foreign tables in Snapshot mode
> レイクベース同期テーブルは、Apache Iceberg およびスナップショットモードの外部テーブルの同期をサポートしています。
August 28, 2025

You can now create synced tables in Snapshot sync mode from Iceberg tables or foreign tables.

See Supported source tables.

> Icebergテーブルやフォーリンテーブルから、スナップショット同期モードでデータベースを同期させることができるようになりました。 サポートされているソーステーブルについては、こちらのドキュメントをご確認ください。
External MCP servers are in Beta
> 外部 MCP サーバーはベータ版です。
August 28, 2025

Users can now connect Databricks to external Model Context Protocol (MCP) servers. This allows agents to access tools outside of Databricks.

See Model Context Protocol (MCP) on Databricks.

> ユーザーは今Databricksを外部のModel Context Protocol (MCP)サーバーに接続できるようになりました。これにより、エージェントはDatabricks以外のツールにアクセスできるようになります。 DatabricksのModel Context Protocol (MCP)をご覧ください。

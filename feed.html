Databricks Docs Updates

Dashboard pivot tables, gridlines, and drill-through enhancements
> ダッシュボードの pivot テーブル、グリッドライン、およびドリルスルーの機能強化
October 16, 2025
  • Dynamic text functions in pivot tables: Pivot tables now support contains(), startswith(), and endswith() functions for conditional formatting. See Conditional formatting with text functions.
  • Gridline toggle: Charts now support toggling gridlines on and off. See Gridlines
  • Dashboard tags and certification in Public Preview: You can now add tags to dashboards and Genie spaces to improve organization, search, and discovery across your workspace. Tagged items are indexed for search, making it easier for users to find relevant dashboards and spaces quickly.
  • Drill-through without target filters: Drill-through now filters any visualization based on the same dataset as the source selection without an explicit target filter. If a target filter exists, it is still applied. See Use drill through to filter data.
  • Fixed discarded dashboard API error: APIs now return 404 NOT_FOUND instead of 403 PERMISSION_DENIED when accessing dashboards in the trash.
>
  • ピボットテーブルのダイナミックテキスト関数: ピボットテーブルで条件付き書式設定に contains()、startswith()、endswith() 関数がサポートされるようになりました。テキスト関数による条件付き書式設定をご覧ください。
  • グリッド線のトグル: チャートでグリッド線のオン/オフの切り替えがサポートされるようになりました。グリッド線をご覧ください。
  • ダッシュボードタグと認証の公開プレビュー: ワークスペース全体のオーガナイゼーション、検索、発見を改善するためにダッシュボードとGenieスペースにタグを追加できるようになりました。タグ付きの項目は検索対象にインデックスされるため、ユーザーが関連するダッシュボードとスペースをすばやく見つけやすくなります。
  • ターゲットフィルターなしのドリルスルー: ドリルスルーは、ソース選択と同じデータセットに基づいて任意の視覚化をフィルタリングするようになりました。ターゲットフィルターが存在する場合でも、それが引き続き適用されます。ドリルスルーを使ってデータをフィルタリングするをご覧ください。
  • 破棄されたダッシュボードのAPIエラーを修正: ゴミ箱のダッシュボードにアクセスする際、APIは 403 PERMISSION_DENIED ではなく 404 NOT_FOUND を返すようになりました。
Genie API updates, benchmark explanations, and JOIN improvements
> ジーニーAPI更新、ベンチマーク説明、JOINの改善
October 16, 2025
  • JOIN relationship discovery improvements: You can now sort JOIN relationships and SQL snippet definitions for easier management.
  • API endpoints moving to Public Preview: The following Genie API endpoints are moving from Beta to Public Preview: list conversation messages, delete conversation message, and send thumbs up/down feedback.
  • Benchmark result explanations: Benchmark results now include an explanation for why a result was rated incorrect.
  • SQL execution result summaries: Genie now automatically generates summaries of SQL execution results to make responses more natural and easier to interpret.
  • SQL measures in knowledge stores: Users can now add SQL measures to a space's knowledge store, along with filters and dimensions, to guide Genie in generating specific SQL expressions. See Build a knowledge store for more reliable Genie spaces.
  • Fixed permissions messaging: Messaging for users with CAN RUN or CAN VIEW permissions has been corrected to accurately reflect their capabilities in the monitoring and data tabs.
  • Workspace asset tagging support: Genie spaces are now included in the workspace asset tagging public preview.
>
  • JOIN関係の検出機能の改善: JOIN関係およびSQL断片の定義を簡単に管理できるようになりました。
  • APIエンドポイントがパブリックプレビューに移行: 以下のGenieAPIエンドポイントがベータから公開プレビューに移行しました: 会話メッセージの一覧取得、会話メッセージの削除、感謝または非難のフィードバックの送信。
  • ベンチマーク結果の説明: ベンチマーク結果に、結果が正しくない理由の説明が含まれるようになりました。
  • SQL実行結果の概要: Genieが自動的にSQL実行結果の概要を生成するようになり、レスポンスがより自然で解釈しやすくなりました。
  • ナレッジストアでのSQL指標: ユーザーは、フィルターや次元と一緒に、ナレッジストアにSQL指標を追加して、Genieが特定のSQL式を生成するのを支援できるようになりました。詳細はナレッジストアの構築でより信頼性の高いGenieスペースを作成するをご覧ください。
  • 権限メッセージの修正: CAN RUNまたはCAN VIEW権限を持つユーザーに対するメッセージが、モニタリングやデータタブでの機能を正確に反映するよう修正されました。
  • ワークスペースアセットタギングのサポート: Genieスペースがワークスペースアセットタギングのパブリックプレビューに含まれるようになりました。
Dashboard and Genie spaces tagging (Public Preview)
> ダッシュボードとジーニースペースのタグ付け(パブリックプレビュー)
October 16, 2025

You can now add tags to dashboards and Genie spaces to improve organization, search, and discovery across your workspace. Tagged items are indexed for search, making it easier for users to find relevant dashboards and spaces quickly.

See Manage dashboard tags and Add tags.

Databricks Asset Bundles in the workspace is GA
> データブリックス ワークスペースのアセットバンドルがGAになりました。
October 16, 2025

Databricks Asset Bundles in the workspace is now generally available (GA). This feature allows you to collaborate with other users in your organization to edit, commit, test, and deploy bundle updates through the UI.

See Collaborate on bundles in the workspace.

> ワークスペース内のDatabricks Asset Bundlesが一般に公開されました。この機能により、組織の他のユーザーと協力して、UIを通じてバンドルの更新を編集、コミット、テスト、およびデプロイすることができます。 ワークスペース内でバンドルを共同作業するには、こちらをご覧ください。

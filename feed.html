Databricks Docs Updates

Databricks SQL version 2025.25 is now available in Preview
> データブリックス SQL バージョン 2025.25がプレビューで利用可能になりました。
August 14, 2025

Databricks SQL version 2025.25 is now available in the Preview channel. Review the following section to learn about new features and behavioral changes.

Expanded spatial SQL expressions and GEOMETRY and GEOGRAPHY data types

You can now store geospatial data in built-in GEOMETRY and GEOGRAPHY columns for improved performance of spatial queries. This release adds more than 80 new spatial SQL expressions, including functions for importing, exporting, measuring, constructing, editing, validating, transforming, and determining topological relationships with spatial joins. See ST geospatial functions, GEOGRAPHY type, and GEOMETRY type.

Support for schema and catalog level default collation

You can now set a default collation for schemas and catalogs. This allows you to define a collation that applies to all objects created within the schema or catalog, ensuring consistent collation behavior across your data.

Better handling of JSON options with VARIANT

The from_json and to_json functions now correctly apply JSON options when working with top-level VARIANT schemas. This ensures consistent behavior with other supported data types.

Support for TIMESTAMP WITHOUT TIME ZONE syntax

You can now specify TIMESTAMP WITHOUT TIME ZONE instead of TIMESTAMP_NTZ. This change improves compatibility with the SQL Standard.

Resolved subquery correlation issue

Databricks no longer incorrectly correlates semantically equal aggregate expressions between a subquery and its outer query. Previously, this could lead to incorrect query results.

Error thrown for invalid CHECK constraints

Databricks now throws an AnalysisException if a CHECK constraint expression cannot be resolved during constraint validation.

Stricter rules for stream-stream joins in append mode

Databricks now disallows streaming queries in append mode that use a stream-stream join followed by window aggregation, unless watermarks are defined on both sides. Queries without proper watermarks can produce non-final results, violating append mode guarantees.

Databricks SQLバージョン2025.25がプレビューチャンネルで利用可能となりました。新機能とベヘイビアの変更点を以下に確認してください。

拡張された空間SQL式とGEOMETRYおよびGEOGRAPHYデータ型

地理空間データを組み込みのGEOMETRYおよびGEOGRAPHY列に保存し、空間クエリのパフォーマンスを向上させることができます。このリリースでは、データのインポート、エクスポート、測定、構築、編集、検証、変換、空間結合での位相関係の判断など、80を超える新しい空間SQL式が追加されています。ST地理空間関数、GEOGRAPHY型、GEOMETRY型をご確認ください。

スキーマおよびカタログレベルの既定のコレーション設定

スキーマやカタログの既定のコレーションを設定できるようになりました。これによりスキーマやカタログ内で作成されるすべてのオブジェクトに一貫したコレーション動作を定義できます。

VARIANTでのJSONオプションの改善

from_jsonおよびto_json関数が、トップレベルのVARIANTスキーマでJSONオプションを正しく適用するようになりました。これにより、他のサポートされているデータ型との振る舞いの一貫性が確保されます。

TIMESTAMP WITHOUT TIME ZONE構文のサポート

TIMESTAMP_NTZの代わりにTIMESTAMP WITHOUT TIME ZONEを指定できるようになりました。この変更によりSQL Standardとの互換性が向上しました。

サブクエリの相関の問題を解決

Databricksは、サブクエリと外部クエリの間で意味的に等しい集計式を誤って相関させなくなりました。これにより、誤ったクエリ結果が生成されることがなくなりました。

CHECK制約の無効化に対するエラー

Databricksは、CHECK制約式が制約検証中に解決できない場合にAnalysisExceptionをスローするようになりました。

アペンドモードでのストリーム-ストリーム結合の厳格なルール

Databricksは、ウォーターマークが両側に定義されていない限り、ストリームクエリにアペンドモードでストリーム-ストリーム結合と後続のウィンドウ集計を使用することを禁止するようになりました。適切なウォーターマーク

Dashboard table visualization and chart improvements
> ダッシュボードのテーブル表示とチャートの改善
August 14, 2025

  • Indicator for calculated measures in table editor: Added an indicator in the table visualization editor to show when calculated measures are not displayed in the table.

  • Fixed transparent headers in light mode: Resolved a bug where table visualization headers appeared transparent in light mode.

  • Fixed pivot table encoding width retention: Resolved an issue where custom encoding widths in pivot table visualizations were not retained after subsequent edits.

  • Measure fields appear before dimension fields: In the visualization field picker, measure fields now appear before dimension fields.

  • Funnel chart support in AI/BI dashboards: Funnel charts are now supported in AI/BI dashboards. See Funnel chart.

>

  • テーブルエディターの計算メトリックの表示: テーブルビジュアライゼーションエディターに、計算メトリックがテーブルに表示されていない場合を示すインジケーターを追加しました。

  • ライトモードでのテーブルヘッダーの修正: ライトモードでテーブルビジュアライゼーションのヘッダーが透明になる不具合を修正しました。

  • ピボットテーブルのエンコーディング幅の保持: ピボットテーブルビジュアライゼーションで、カスタムのエンコーディング幅が後続の編集後も保持されるよう修正しました。

  • メトリックフィールドが前に表示されるようになりました: ビジュアライゼーションフィールドピッカーで、メトリックフィールドがディメンションフィールドの前に表示されるようになりました。

  • AI/BIダッシュボードでのファネルチャートのサポート: ファネルチャートがAI/BIダッシュボードでサポートされるようになりました。詳細はファネルチャートをご覧ください。

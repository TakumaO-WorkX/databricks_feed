Time travel and VACUUM behavioral changes for Unity Catalog managed tables
> 時間旅行とVACUUMテーブル変更の行動Unity Catalogの管理
December 19, 2025

In January 2026, the time travel and VACUUM behavioral changes introduced in Databricks Runtime 18.0 will extend to serverless compute, Databricks SQL, and Databricks Runtime 12.2 and above for Unity Catalog managed tables.

These changes include:

  • Time travel queries are blocked if they exceed delta.deletedFileRetentionDuration.
  • delta.logRetentionDuration must be greater than or equal to delta.deletedFileRetentionDuration.
> 2026年1月、Databricks Runtime 18.0で導入されたタイムトラベルとVACUUMの動作変更は、Unity Catalog管理テーブルに対してサーバーレスコンピューティング、Databricks SQL、およびDatabricks Runtime 12.2以降にも拡大されます。 これらの変更には以下が含まれます: - タイムトラベルクエリは、delta.deletedFileRetentionDurationを超えた場合にブロックされます。 - delta.logRetentionDurationはdelta.deletedFileRetentionDuration以上でなければなりません。
Dashboard visualization enhancements and fixes
> ダッシュボードの視覚化の改善と修正
December 11, 2025
  • Sort by hidden measure fields: Users can now sort charts by a different measure field that does not visually appear on the bar chart. See Format axis settings.
  • Map legend position control: Authors can now change map legend positions.
  • Safari cross-filtering fix: Fixed an issue where cross-filtering was not working in Safari.
  • Counter visualization text: Removed automatic "vs" text from counter visualizations.
  • Range filter spacing: Added spacing between minimum and maximum values in range filters when the slider bar is hidden.
>
  • 非表示のメジャーフィールドでソートする: ユーザーは、バーチャートに視覚的には表示されないメジャーフィールドでチャートをソートできるようになりました。軸の書式設定をご覧ください。
  • 地図凡例の位置コントロール: 作成者は地図の凡例の位置を変更できるようになりました。
  • Safariのクロスフィルタリングの修正: Safariでクロスフィルタリングが機能しない問題を修正しました。
  • カウンター可視化のテキスト: カウンター可視化から自動的に挿入される「vs」のテキストを削除しました。
  • 範囲フィルターの間隔: スライダーバーが非表示の場合、範囲フィルターの最小値と最大値の間に間隔を追加しました。
Dashboard subscriptions to Microsoft Teams channels (Public Preview)
> Microsoft Teamsチャンネルへのダッシュボードサブスクリプション(パブリックプレビュー)
December 11, 2025

AI/BI dashboards now support sending scheduled snapshots to Microsoft Teams channels. Teams can receive dashboard updates directly in their Teams workspace, including a PNG image snapshot, a direct link to open the dashboard in Databricks, and a PDF attachment. See Configure Microsoft Teams notifications.

> AIおよびBIダッシュボードでは、Microsoft Teamsチャネルに定期的なスナップショットを送信することがサポートされています。Teamsでは、ダッシュボードの更新情報を直接Teamsワークスペースで受け取れるようになりました。これには、PNGイメージのスナップショット、Databricksでダッシュボードを開くためのダイレクトリンク、PDFの添付ファイルが含まれています。詳細については、「Configure Microsoft Teams notifications」をご覧ください。

Genie Research Agent architecture update (Beta)
> ジーニー リサーチ エージェント アーキテクチャ アップデート (ベータ)
December 11, 2025
  • Genie Research Agent architecture update: Research Agent (Beta) now has a single reasoning agent that generates hypotheses and SQL. This helps Research Agent follow instructions more effectively, produce more concise reports, and deliver answers with lower latency.
>
  • ジーニー・リサーチ・エージェントのアーキテクチャアップデート:リサーチ・エージェント(ベータ版)には、仮説と SQLを生成する単一の推論エージェントが追加されました。これにより、リサーチ・エージェントは指示に従って更に効果的に機能し、より簡潔なレポートを作成し、遅延の少ない回答を提供することができます。

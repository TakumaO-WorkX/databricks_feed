AI agents: Authorize on-behalf-of-user Public Preview
> AI エージェント: ユーザーの代理授権パブリックプレビュー
September 02, 2025

AI agents deployed to Model Serving endpoints can use on-behalf-of-user authorization. This lets an agent act as the Databricks user who runs the query for added security and fine-grained access to sensitive data.

> モデル提供エンドポイントに導入されたAIエージェントは、代理ユーザー認証を使用できます。これにより、エージェントが機密データへのきめ細かなアクセスと追加のセキュリティを備えた状態で、クエリを実行するDatabricsユーザーとして機能できます。
Account SCIM 2.0 updates
> アカウントSCIM 2.0の更新
August 29, 2025

To enhance stability and support future growth, Databricks has updated the Account SCIM API for identity management as follows:

  • Calling GET with filter params filter=displayName eq value_without_quotes results in a syntax error. To prevent this error, use quotation marks to wrap the value (for example, filter=displayName eq "value_with_quotes").

  • Calling GET /api/2.0/accounts/{account_id}/scim/v2/Groups no longer returns members. Instead, iterate through get group details to get membership information. See get group details.

  • Calling PATCH /api/2.0/accounts/{account_id}/scim/v2/Groups/{id} returns a 204 response instead of a 200 response.

These changes apply to new accounts and will be grandfathered into existing accounts. Current integrations will continue to work without disruption.

> データブリックスは、アイデンティティ管理のためのAccount SCIM APIを次のように更新しました: - `GET with filter params filter=displayName eq value_without_quotes`を呼び出すとシンタックスエラーが発生します。このエラーを防ぐには、値を引用符で囲む必要があります(例: `filter=displayName eq "value_with_quotes"`)。 - `GET /api/2.0/accounts/{account_id}/scim/v2/Groups`を呼び出しても、メンバーは返されません。代わりに、`get group details`を繰り返し呼び出して、メンバーシップ情報を取得してください。詳細については、[get group details](https://docs.databricks.com/api/account/accountgroups/get)を参照してください。 - `PATCH /api/2.0/accounts/{account_id}/scim/v2/Groups/{id}`を呼び出すと、200レスポンスではなく204レスポンスが返されます。 これらの変更は新しいアカウントに適用され、既存のアカウントにも反映されます。現在のインテグレーションは中断することなく継続して動作します。
Track and navigate notebook runs with the new cell execution minimap
> セル実行ミニマップを使用した新しいノートブックの追跡とナビゲーション
August 28, 2025

Use the cell execution minimap to track your notebook’s progress at a glance. The minimap appears in the right margin and shows each cell’s execution state (skipped, queued, running, success, or error). Hover to see cell details, or click to jump directly to a cell.

For information about using the cell execution minimap, see Navigate the Databricks notebook and file editor.

> ノートブックの進行状況をすばやく把握するために、セル実行ミニマップを使用してください。ミニマップは右のマージンに表示され、各セルの実行状態(スキップ、待機中、実行中、成功、またはエラー)を示します。セルの詳細を確認するにはカーソルを合わせ、直接セルに移動するにはクリックします。 セル実行ミニマップの使用方法については、「Databricksノートブックとファイルエディタの操作」をご覧ください。
Admins can now manage a workspace's serverless base environments (Public Preview)
> サーバーレスベース環境(パブリックプレビュー)のワークスペースを管理できるようになりました。
August 28, 2025

Base environments are custom environment specifications for serverless notebooks that define a serverless environment version and a set of dependencies.

Workspace admins can now create and manage the base environments available in their workspace. They can also set a default base environment for all new serverless notebooks. This allows workspace users to quickly start working in a consistent, cached environment.

See Manage serverless base environments.

> ベースの環境は、サーバーレスノートブックのためのカスタム環境の仕様で、サーバーレス環境のバージョンと一連の依存関係を定義します。 ワークスペースの管理者は、ワークスペースで利用可能なベースの環境を作成および管理できるようになりました。また、新しいサーバーレスノートブックのデフォルトのベース環境を設定することもできます。これにより、ワークスペースのユーザーは、一貫性のある、キャッシュされた環境で素早く作業を開始できるようになります。 サーバーレスのベース環境の管理の詳細については、こちらをご覧ください。

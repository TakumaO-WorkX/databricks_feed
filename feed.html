Databricks Docs Updates

Dashboard visualization enhancements
> ダッシュボードの可視化の向上
November 27, 2025
  • Sort by hidden measure fields: Users can now sort charts by a different measure field that does not visually appear on the bar chart. See Format axis settings.
  • Pivoted data downloads: Dashboard visualizations now support downloading data in a pivoted shape.
  • Improved pivot table performance: Pivot table headers now render significantly faster with reduced scroll lag.
>
  • 非表示メジャーフィールドによる並べ替え: ユーザーは、バーチャートに視覚的に表示されない別のメジャーフィールドによってチャートを並べ替えることができます。詳細は軸の書式設定をご覧ください。
  • ピボットデータのダウンロード: ダッシュボードの視覚化では、ピボット形式でデータをダウンロードできるようになりました。
  • ピボットテーブルのパフォーマンス向上: ピボットテーブルのヘッダーのレンダリングが大幅に高速化され、スクロールラグが減少しました。
Genie space limits and Research Agent improvements
> 空間の制限とリサーチ・エージェントの改善
November 27, 2025
  • Increased SQL snippets and JOIN relationship limit: The limit for SQL snippets and JOIN relationships in a Genie space has been increased to 200.
  • Research Agent starter questions: Research Agent (Beta) now displays starter questions to help users begin their data exploration.
  • Auto-suggested benchmarks: Genie now automatically suggests benchmarks based on the space context to help authors build evaluation sets.
  • Fixed knowledge store permissions: Resolved an issue where users with CAN EDIT permissions couldn't manage Genie knowledge store column descriptions and value dictionaries.
>
  • SQL スニペットと JOIN 関係の制限の増加: Genie スペースの SQL スニペットとJOIN関係の制限が 200 に増加しました。
  • リサーチ エージェントの事前質問: リサーチ エージェント (ベータ版) では、ユーザーがデータ探索を始めるのを支援するために事前質問が表示されるようになりました。
  • 自動提案されるベンチマーク: Genie では、著者がエバリュエーション セットを構築するのを支援するため、スペースのコンテキストに基づいて自動的にベンチマークが提案されるようになりました。
  • 知識ストアのアクセス許可の修正: CAN EDIT 権限を持つユーザーでも Genie の知識ストアの列の説明や値の辞書を管理できないという問題が解決されました。
Lakeflow Spark Declarative Pipelines stream progress metrics are generally available

Lakeflow Spark Declarative Pipelines support for querying the event log for metrics about the progress of a stream is now generally available. See Monitor pipeline streaming metrics.

Lakeflow Spark 宣言的パイプラインは、ストリームの進捗に関するメトリクスを取得するためのイベントログクエリをサポートするようになりました。詳細はパイプラインのストリーミングメトリクスを監視するをご覧ください。

Databricks Assistant Agent Mode preview
> データブリックス・アシスタント・エージェントモードのプレビュー
November 21, 2025

The Databricks Assistant Agent Mode preview will soon be enabled by default for most customers.

  • The agent can automate multiple steps. From a single prompt, it can retrieve relevant assets, generate and run code, fix errors automatically, and visualize results. It adds the ability to sample data and cell outputs to provide better results.
  • The Assistant in Agent Mode will choose between Azure OpenAI or Anthropic on Databricks (uses endpoints hosted by Databricks Inc. in AWS within the Databricks security perimeter), and is only available when the partner-powered AI features setting is enabled.
  • Admins can disable the preview if needed until the feature reaches General Availability.

See Use the Data Science Agent, the blog post, and Partner-powered AI features.

>

Databricks アシスタント エージェントモードのプレビューは、ほとんどの顧客に対して間もなく既定で有効になります。

  • エージェントは複数のステップを自動化できます。1つのプロンプトから、関連するアセットを取得し、コードを生成して実行し、エラーを自動的に修正し、結果を視覚化します。データのサンプリングとセル出力の追加により、より良い結果を提供することができます。
  • エージェントモードのアシスタントは、Databricks上のAzure OpenAIまたはAnthropic(Databricks Inc.によってAWS内のDatabricks セキュリティ境界にホストされたエンドポイントを使用)のいずれかを選択し、パートナー提供のAI機能設定が有効な場合にのみ利用可能です。
  • 管理者は、機能が一般提供となるまでプレビューを必要に応じて無効にできます。

データ科学エージェントの使用、ブログ記事、パートナー提供のAI機能を参照してください。

Configure full refresh behavior for managed database connectors
> データベースコネクタの完全リフレッシュ動作の設定
November 21, 2025

You can now schedule when full refresh snapshots occur and enable automatic full refresh to recover from unsupported schema changes. This applies to managed database connectors like SQL Server in Lakeflow Connect. See Configure full refresh behavior for database connectors.

> データベースコネクタの完全リフレッシュのスケジュールを設定し、サポートされていないスキーマ変更からの復旧のために自動完全リフレッシュを有効にできます。これはLakeflow ConnectのSQL Serverなどの管理データベースコネクタに適用されます。詳細はデータベースコネクタの完全リフレッシュの動作を設定するをご覧ください。

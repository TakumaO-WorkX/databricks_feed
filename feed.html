Databricks Docs Updates

Databricks One is generally available
> データブリックス ワンは一般に利用可能になりました。
January 20, 2026

Databricks One is now generally available. Databricks One is a simplified user interface designed for business users, providing a single, intuitive entry point to interact with data and AI in Databricks without requiring technical knowledge of compute resources, queries, models, or notebooks. Using Databricks One, business users can view and interact with dashboards, ask data questions in natural language using Genie, and use custom-built Databricks Apps that combine analytics, AI, and workflows. See What is Databricks One?.

> Databricks Oneは現在一般に利用可能です。Databricks Oneは、コンピューティングリソース、クエリ、モデル、ノートブックの技術的な知識を必要とせずに、DatabricksのデータおよびAIとやり取りできる単一の直感的なエントリポイントを提供するように設計された簡略化されたユーザーインターフェイスです。Databricks Oneを使用すると、ビジネスユーザーはダッシュボードの表示と操作、Genieを使用した自然言語によるデータ問合せ、分析、AI、ワークフローを組み合わせたDatabricksアプリの利用ができます。Databricks Oneとは何ですか?をご覧ください。
You can now use custom base environments for Python, Python Wheels, and notebook tasks in serverless jobs
> Python、Python Wheels、ノートブックタスクにカスタムベース環境を使用できるようになりました。
January 20, 2026

Serverless jobs now support custom base environments defined with YAML files for Python, Python wheel, and notebook tasks. For notebook tasks, you can either select a custom base environment in the job’s environment configuration or use the notebook’s own environment settings, which support both workspace environments and custom base environments.

For more information, see Manage serverless base environments.

> サーバーレスジョブは、Python、Pythonホイール、およびノートブークタスクのYAMLファイルで定義されたカスタム基本環境をサポートするようになりました。ノートブークタスクの場合、ジョブの環境設定でカスタム基本環境を選択するか、ワークスペース環境とカスタム基本環境の両方をサポートするノートブークの独自の環境設定を使用できます。 詳細については、「サーバーレス基本環境の管理」をご覧ください。
Level of detail expressions for custom calculations
> レベルの詳細表現をカスタム計算に使用します
January 19, 2026

Level of detail expressions let you control aggregation granularity independently of the dimensions in your visualizations. Use fixed level of detail expressions to aggregate over specific dimensions regardless of visualization groupings, or use coarser level of detail expressions to aggregate at a higher level by excluding dimensions. Common use cases include calculating percentages of total, comparing values to dataset-wide aggregates, and creating cohort-level metrics. See Level of detail expressions.

>

詳細レベルの式を使うと、視覚化に含まれる次元とは無関係に、集計の粒度を制御できます。固定の詳細レベルの式を使うと、視覚化のグループ化に関係なく、特定の次元で集計できます。あるいは、より粗い詳細レベルの式を使うと、次元を除外して、より高いレベルで集計できます。一般的な使用例には、合計に対するパーセンテージの計算、データセット全体の集計値との比較、コホート単位のメトリクスの作成などがあります。詳細レベルの式を参照してください。

Value sampling features renamed
> 価値サンプリング機能の名称変更
January 16, 2026

The value sampling features in Genie have been renamed to better describe their function:

  • Prompt matching (previously value sampling)
  • Format assistance (previously example values)
  • Entity matching (previously value dictionaries)
>

Genieの値サンプリング機能は、その機能をより適切に説明するために名称が変更されました:

  • プロンプトマッチング(以前の値サンプリング)
  • フォーマットアシスト(以前の例示値)
  • エンティティマッチング(以前の値辞書)
Dashboard navigation, pivot tables, and text widget improvements
> ダッシュボードナビゲーション、ピボットテーブル、テキストウィジェットの改善
January 15, 2026
  • Dashboard page navigation scroll bar: The horizontal scroll bar to navigate between dashboard pages is now thicker.
  • Dashboard title character limit: Dashboard titles now have more horizontal space before truncating.
  • Pivot table header height: Pivot table header height can be configured to enable wrapping for multi-row heights.
  • Text widget markdown tables: Text widgets now support markdown table syntax.
>
  • ダッシュボードページのナビゲーションスクロールバー: ダッシュボードページを移動するための水平スクロールバーが太くなりました。
  • ダッシュボードのタイトル文字数制限: ダッシュボードのタイトルは、切り詰められる前に水平方向によりスペースが取れるようになりました。
  • ピボットテーブルのヘッダー高さ: ピボットテーブルのヘッダー高さを設定して、複数行の高さに折り返すことができるようになりました。
  • テキストウィジェットのMarkdownテーブル: テキストウィジェットでMarkdownのテーブル構文がサポートされるようになりました。

Databricks Docs Updates

Create external Delta tables from third-party clients (Public Preview)
> Delta湖からサードパーティクライアントに外部Deltaテーブルを作成する(パブリックプレビュー)
August 08, 2025

You can now create Unity Catalog external tables backed by Delta Lake from external clients and systems, such as Apache Spark.

See Create external Delta tables from external clients.

>

Unity Catalog外部テーブルをDelta Lakeで作成し、Apache Sparkなどの外部クライアントやシステムから使用できるようになりました。

詳細は外部クライアントからDelta外部テーブルを作成するをご覧ください。

Path credential vending (Public Preview)
> パスクレデンシャル販売（パブリックプレビュー）
August 08, 2025

You can now use path credential vending to grant short-lived credentials to external locations in your Unity Catalog metastore. See Unity Catalog credential vending for external system access.

> パスのクレデンシャル提供を使用して、Unity Catalogメタストアの外部の場所に短期のクレデンシャルを付与できるようになりました。外部システムアクセスのためのUnity Catalogクレデンシャル提供を参照してください。
Fixed timeout handling for materialized views and streaming tables
> 固定タイムアウトハンドリングのためのマテリアライズドビューとストリーミングテーブル
August 07, 2025

New timeout behavior for materialized views and streaming tables created in Databricks SQL:

  • Materialized views and streaming tables created after August 7, 2025 will have the warehouse timeout automatically applied.
  • For materialized views and streaming tables created before August 7, 2025, run CREATE OR REFRESH to synchronize the timeout setting with the timeout configuration of the warehouse.
  • All materialized views and streaming tables now have a default timeout of two days.
>

Databricks SQLで作成された具現化ビューとストリーミングテーブルの新しいタイムアウト動作:

  • 2025年8月7日以降に作成された具現化ビューとストリーミングテーブルには、自動的にウェアハウスのタイムアウトが適用されます。
  • 2025年8月7日より前に作成された具現化ビューとストリーミングテーブルの場合は、CREATE OR REFRESHを実行してタイムアウト設定をウェアハウスの設定と同期させてください。
  • すべての具現化ビューとストリーミングテーブルのデフォルトのタイムアウトは2日間です。
Use OpenAI GPT OSS models for batch inference and structured outputs
> OpenAIのGPT OSSモデルを使ってバッチ推論と構造化出力を行う
August 07, 2025

The Databricks-hosted foundation models, OpenAI GPT OSS 120B and GPT OSS 20B are now optimized for AI Functions, which means you can perform batch inference using these models and AI Functions like ai_query(). Inference workloads that use these models also now support specifying structured outputs.

> オープン AIの GPT OSS 120Bと GPT OSS 20Bは、Databricksホストのファウンデーションモデルであり、AI関数に最適化されています。つまり、これらのモデルを使って「ai_query()」などのAI関数を使ってバッチ推論を行うことができます。また、これらのモデルを使った推論ワークロードでは、構造化出力を指定することもできるようになりました。
Serverless compute is now available in ap-southeast-3
> サーバーレスコンピューティングが現在 ap-southeast-3 で利用可能です。
August 06, 2025

Serverless compute for notebooks, workflows, and Lakeflow Declarative Pipelines is now available in the Asia Pacific (Jakarta) region (ap-southeast-3). See Features with limited regional availability.

>

ノートブック、ワークフロー、Lakeflow 宣言的パイプラインのサーバーレスコンピューティングが、アジア太平洋(ジャカルタ)リージョン(ap-southeast-3)で利用可能になりました。リージョンの限定的な機能を参照してください。

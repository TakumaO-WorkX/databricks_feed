Databricks Docs Updates

Dashboard enhancements and fixes
> ダッシュボードの機能改善と修正
December 18, 2025
  • Boolean alias formatting: Boolean values now support alias formatting, allowing users to replace "true" and "false" with custom strings.
  • Compact global filters: Reduced the size of global filters to save space.
  • Relative date range options: Relative date range filters now include yesterday, last week, last month, and last year.
  • Filter widget background fix: Fixed an issue where a gray background appeared on some filter widgets.
  • Filter reset button fix: Fixed an issue where the "Reset to default" button appeared on the active filter bar when all filters showed default values.
  • Custom chart labels: You can now choose a custom field for chart labels. See Value labels.
  • Sortable filter dropdown options: Authors can now sort filter dropdown options by another field.
  • Multi-select filter paste: Viewers can now copy a column of values from a spreadsheet and paste them into a multi-select filter.
  • Chart tooltip overflow indicator: Chart tooltips with many series now indicate when there are more series.
  • Axis label spacing fix: Fixed an issue where long axis labels took up too much plotting area.
  • Agent-created widget consistency: Agent-created widgets now work consistently across edit and published dashboards.
  • Mark label angle fix: Fixed mark label angle for charts with multiple Y axes without an X axis.
>
  • ブーリアンエイリアスの書式設定: ブーリアン値がカスタムの文字列で"true"と"false"を置き換えることができるようになりました。
  • コンパクトなグローバルフィルター: グローバルフィルターのサイズが小さくなり、スペースを節約できるようになりました。
  • 相対的な日付範囲オプション: 相対的な日付範囲フィルターに昨日、先週、先月、昨年が含まれるようになりました。
  • フィルターウィジェットの背景の修正: 一部のフィルターウィジェットで灰色の背景が表示される問題が修正されました。
  • フィルターリセットボタンの修正: すべてのフィルターがデフォルト値を表示している時に、アクティブなフィルターバーに"デフォルトにリセット"ボタンが表示される問題が修正されました。
  • カスタムチャートラベル: チャートラベルにカスタムのフィールドを選択できるようになりました。Value labelsを参照してください。
  • ソート可能なフィルタードロップダウンオプション: 作成者がフィルターのドロップダウンオプションを他のフィールドでソートできるようになりました。
  • マルチ選択フィルターのペースト: 閲覧者がスプレッドシートの列の値をコピーしてマルチ選択フィルターに貼り付けられるようになりました。
  • チャートツールチップのオーバーフロー表示: 多数のシリーズがあるチャートのツールチップで、他のシリーズがあることを示すようになりました。
  • 軸ラベルの間隔の修正: 長い軸ラベルが描画領域を占有しすぎる問題が修正されました。
  • エージェント作成のウィジェットの一貫性: エージェント作成のウィジェットが編集と公開されたダッシュボードで一貫して動作するようになりました。
  • マークラベルの角度の修正: X軸のないチャートで複数のY軸があるときのマークラベルの角度が修正されました。
Genie enhancements and fixes
> 遺伝子工学の改良と修正
December 18, 2025
  • Numeric column formatting: Numeric columns in tables are now formatted more effectively, including IDs, ZIP codes, and currency.
  • File upload fix for CAN MANAGE users: Fixed an issue where CAN MANAGE users couldn't use the upload file feature.
  • Value dictionary indexing improvements: Improved value dictionary indexing for better coverage and recall on user prompts.
>
  • 数値列の書式設定: テーブルの数値列が、ID、郵便番号、通貨など、より効果的に書式設定されるようになりました。
  • CAN MANAGE ユーザーのファイルアップロードの修正: CAN MANAGE ユーザーがファイルアップロード機能を使えない問題を修正しました。
  • 値辞書のインデックス化の改善: ユーザープロンプトの対応範囲と再現性を高めるため、値辞書のインデックス化を改善しました。
Public URLs for ODBC driver downloads will be disabled
> 公開URLのODBCドライバーのダウンロードは無効になります。
December 18, 2025

In an upcoming release, public URLs for automated Apache Spark ODBC driver downloads will be disabled. After this change, all driver downloads, including automated processes, will require authentication. This will break automated processes that download Apache Spark ODBC drivers using direct public links without authentication.

>

今後のリリースで、自動化されたApache Spark ODBCドライバのダウンロード用の公開URLが無効になります。この変更以降、すべてのドライバのダウンロード、自動プロセスを含め、認証が必要になります。これにより、認証なしで直接公開リンクを使ってApache Spark ODBCドライバをダウンロードする自動プロセスが機能しなくなります。

Lakebase Autoscaling metrics dashboard
> レイクベースの自動スケーリングメトリクスダッシュボード
December 18, 2025

Lakebase Autoscaling (Public Preview) now includes a Metrics dashboard for monitoring system and database metrics. See Metrics.

>

レイクベースの自動スケーリング(パブリックプレビュー)に、システムおよびデータベースメトリクスを監視するメトリクスダッシュボードが含まれるようになりました。メトリクスをご覧ください。

View latest scheduled notebook job results
> 最新のスケジュールされたノートブックジョブの結果を表示する
December 18, 2025

Databricks notebooks can now show the latest scheduled notebook run directly in your notebook and notebook dashboards. You can also update the notebook with the latest run results.

For more details, see View last successful run and update notebook.

> Databricksのノートブックで、ノートブックとノートブックダッシュボードに直接最新のスケジュールされたノートブックの実行を表示できるようになりました。また、最新の実行結果でノートブックを更新することもできます。 詳細については、「最後の成功した実行を表示してノートブックを更新する」をご覧ください。

Databricks Docs Updates

Databricks Assistant Agent Mode preview
> データブリックス・アシスタントのエージェントモードプレビュー
November 21, 2025

The Databricks Assistant Agent Mode preview will soon be enabled by default for most customers.

  • The agent can automate multiple steps. From a single prompt, it can retrieve relevant assets, generate and run code, fix errors automatically, and visualize results. It adds the ability to sample data and cell outputs to provide better results.
  • The Assistant in Agent Mode will choose between Azure OpenAI or Anthropic on Databricks (uses endpoints hosted by Databricks Inc. in AWS within the Databricks security perimeter), and is only available when the partner-powered AI features setting is enabled.
  • Admins can disable the preview if needed until the feature reaches General Availability.

See Use the Data Science Agent, the blog post, and Partner-powered AI features.

>

Databricks アシスタントエージェントモードのプレビューが間もなくほとんどのお客様にデフォルトで有効になります。

  • エージェントは複数のステップを自動化することができます。単一のプロンプトから、関連するアセットを取得し、コードを生成して実行し、自動的にエラーを修正し、結果を可視化することができます。データのサンプリングとセル出力の機能を追加して、より良い結果を提供できるようになります。
  • エージェントモードのアシスタントは、Azure OpenAI またはAnthropic on Databricks (Databricks Inc. が AWS 内のDatabricks セキュリティ境界でホストしているエンドポイントを使用) を選択し、パートナー提供のAI 機能設定が有効になっている場合にのみ利用できます。
  • 管理者は、機能が一般公開に達するまでプレビューを必要に応じて無効にできます。

データサイエンスエージェントの使用、ブログ記事、およびパートナー提供のAI 機能をご覧ください。

Automate classic workspace creation with AWS IAM temporary delegation
> 作業スペースの自動作成をAWS IAMの一時委任で行う
November 21, 2025

You can now automate the creation of classic workspaces using AWS IAM temporary delegation, which grants Databricks temporary access to provision the required IAM roles, S3 bucket, access policies, and customer-managed VPC in your AWS account. This workspace creation flow replaces the AWS Quickstart template.

See Create a classic workspace.

> AWSIAMの一時的な委任を使用して、必要なIAMロール、S3バケット、アクセスポリシー、およびお客様管理VPCをプロビジョニングするクラシックワークスペースの作成を自動化できるようになりました。このワークスペースの作成フローは、AWSクイックスタートテンプレートに代わるものです。 クラシックワークスペースの作成については、次のリンクを参照してください。
Real-time collaboration in notebook cells, files, and SQL editor
> リアルタイムのノートブックセル、ファイル、およびSQL エディターでのコラボレーション
November 21, 2025

Real-time collaboration is now available in notebook cells, files, and the new SQL editor. Multiple users can edit the same cell simultaneously and can view each other's edits. For more information, see Collaborate using Databricks notebooks.

> ノートブックのセルやファイル、新しいSQLエディターでリアルタイムコラボレーションが利用可能になりました。複数のユーザーが同時に同じセルを編集し、お互いの編集内容を確認できます。詳細については、Databricksノートブックを使ったコラボレーションを参照してください。

Dashboard visualization improvements and bug fixes
> ダッシュボード・ビジュアライゼーションの改善とバグ修正
November 20, 2025
  • Counter visualization improvements: You can now use counters to track and compare metrics over time and show sparklines. To create a counter with a sparkline, select a main value and a date column. See Counter visualization.
  • Configure filter application behavior: Dashboards now support applying multiple filters at the same time alongside the default instant application. See Configure filter application behavior.
  • Fixed data label positioning: Resolved an issue where data labels were sometimes half above and half below bars on bar charts.
  • Fixed label color legibility: Resolved an issue where label color was not legible against all background colors.
>
  • カウンター可視化の改善: カウンターを使って時間の経過と比較に沿ってメトリックを追跡し、スパークラインを表示できるようになりました。メイン値と日付列を選択してカウンターにスパークラインを作成します。カウンター可視化をご覧ください。
  • フィルターの適用方法の設定: ダッシュボードで、デフォルトの即時適用に加えて複数のフィルターを同時に適用できるようになりました。フィルターの適用方法の設定をご覧ください。
  • データラベルの位置の修正: バーチャート上でデータラベルが時々バーの上半分と下半分に分かれて表示されていた問題を解決しました。
  • ラベルの色の見やすさの修正: ラベルの色が背景色によっては見づらくなっていた問題を解決しました。

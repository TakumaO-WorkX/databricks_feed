Chat in Databricks One (Beta)
> チャットInデータブリックスワン(ベータ)
March 13, 2026

Chat in Databricks One is now available in Beta. Chat provides a unified, full-screen interface for talking to Genie. Genie routes your question to the most relevant Genie space or directly answers your question agentically using available data and data assets. See Chat in Databricks One.

>

Databricks Oneのチャットは現在ベータ版で利用可能です。チャットでは、Genieとの対話のためのユニファイドなフルスクリーンのインターフェイスが提供されます。Genieは、利用可能なデータやデータアセットを使って、最も関連性の高いGenieスペースにユーザーの質問をルーティングするか、直接回答します。Databricks Oneのチャットについてをご確認ください。

Some partner-powered AI features are now available in-country for Brazil, Canada, Japan, Korea, Singapore, and the UK
> ブラジル、カナダ、日本、韓国、シンガポール、英国で一部のパートナー駆動型AIの機能が利用できるようになりました。
March 13, 2026

The following features are now available through models hosted in-country in Brazil, Canada, Japan, Korea, Singapore, and the UK:

  • Genie Code chat and cell actions
  • Genie (excluding research agent)
  • AI-generated comments
  • AI/BI dashboards

Workspaces in these countries can use these features with Enforce data processing within workspace Geography for Designated Services enabled and partner-powered AI features enabled. For these features in the listed countries, in-country data routing is enforced, meaning data does not leave the country.

If partner-powered AI features is disabled, Genie Code can use Databricks-hosted models only if Enforce data processing within workspace Geography for Designated Services is disabled.

All other designated services continue to comply with data residency managed by Databricks Geos. Some AI capabilities that depend on models not currently hosted in these countries might not be available when Enforce data processing within workspace Geography for Designated Services is enabled. See Databricks Geos: Data residency for more information on which features require cross-geo processing.

>

以下の機能がブラジル、カナダ、日本、韓国、シンガポール、英国にホストされるモデルから利用可能になりました:

  • Genie Codeチャットとセルのアクション
  • Genie(リサーチエージェントを除く)
  • AI生成コメント
  • AI/BIダッシュボード

これらの国のワークスペースでは、指定サービスのワークスペースエリア内でのデータ処理を強制とパートナー提供のAI機能を有効にすることで、これらの機能を使用できます。これらの国の機能では、国内データルーティングが強制されるため、データが国外に出ることはありません。

パートナー提供のAI機能が無効の場合、Genie Codeは指定サービスのワークスペースエリア内でのデータ処理を強制を無効にすれば、Databricksホストのモデルのみを使用できます。

その他の指定サービスについては、Databricks Geosによる データ保管場所の管理に準拠します。これらの国でホストされていないモデルに依存するAI機能の一部は、指定サービスのワークスペースエリア内でのデータ処理を強制を有効にすると利用できなくなる可能性があります。詳細は、Databricks Geos: データ保管場所を参照してください。

Automatic deletion of empty vector search endpoints
> 自動的な空ベクトル検索エンドポイントの削除
March 13, 2026

Vector search endpoints are now only charged after an index has been created, and 24 hours after the last index is deleted from the endpoint, the endpoint no longer incurs any charges.

For more information, see Mosaic AI Vector Search: Cost management guide.

> ベクターサーチのエンドポイントは、インデックスが作成された後にのみ課金されるようになり、最後のインデックスがエンドポイントから削除されてから24時間経つと、エンドポイントにはもはや課金されません。 詳細については、「Mosaic AI ベクターサーチ:コスト管理ガイド」をご覧ください。
5X-Large SQL warehouse size (Beta)
> 5XLarge SQLウェアハウスサイズ(ベータ)
March 13, 2026

The 5X-Large cluster size for serverless and pro SQL warehouses is now available in Beta. For serverless SQL warehouses, 5X-Large is available in all supported regions. Workspace admins can control access to this feature from the Previews page. See Manage Databricks previews.

For more information on SQL warehouse sizing, see SQL warehouse sizing, scaling, and queuing behavior.

>

サーバーレスおよびプロのSQL ウェアハウスの5X-Large クラスタサイズがベータで利用可能になりました。サーバーレスSQL ウェアハウスでは、5X-Largeが全ての対応リージョンで利用可能です。ワークスペース管理者はプレビューページからこの機能へのアクセスを制御できます。Databricksプレビューの管理をご覧ください。

SQL ウェアハウスのサイジングの詳細については、SQL ウェアハウスのサイジング、スケーリング、およびキューイングの振る舞いをご参照ください。

Dashboard enhancements and improvements
> ダッシュボードの機能強化と改善
March 12, 2026
  • Dashboard embedding for external users is generally available: Embed AI/BI dashboards in external applications without requiring viewers to have Databricks accounts. See What is embedding for external users?.
  • Hide Databricks logo in embedded dashboards: Externally embedded dashboards now support hiding the Databricks logo using the hideDatabricksLogo option. See Hide the Databricks logo.
  • Metric view lineage highlighting: Hovering over metric views in the lineage panel now highlights the visualizations built on those metric views.
  • Certification and deprecation icons in lineage: Data objects in the dashboard lineage panel now show their certification or deprecation status.
  • Clear individual active filters: You can now remove a single filter selection from the active filter bar without resetting all active filters.
  • Mobile layouts: Dashboards now automatically render in a mobile-friendly layout when viewed on small screens. See Mobile layout.
  • Custom subscription email subjects: When configuring a dashboard subscription, authors can now edit the subject line of notification emails sent to subscribers.
  • Accounting-style number formatting: Number fields now support accounting-style formatting, which displays negative values in parentheses and vertically aligns currency symbols.
  • Custom fonts: Dashboard authors can now select custom fonts for dashboards. Fonts must be preinstalled on viewer machines. If a font is not available, dashboards fall back to Databricks default fonts. See Set a custom font.
  • Genie Code for dashboard authoring (Public Preview): Automate multi-step dashboard workflows using natural language prompts to create datasets, visualizations, layouts, and filters. See Use Genie Code for dashboard authoring.

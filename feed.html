Databricks Docs Updates

Use OpenAI GPT OSS models for batch inference and structured outputs
> OpenAIのGPT OSSモデルを使用して、バッチ推論と構造化された出力を行います。
August 07, 2025

The Databricks-hosted foundation models, OpenAI GPT OSS 120B and GPT OSS 20B are now optimized for AI Functions, which means you can perform batch inference using these models and AI Functions like ai_query(). Inference workloads that use these models also now support specifying structured outputs.

>

Databricksがホストするファウンデーションモデル、OpenAI GPT OSS 120BおよびGPT OSS 20Bは、AI Functionsに最適化されました。これは、これらのモデルとAI Functionsのようなai_query()を使用してバッチ推論を実行できることを意味します。これらのモデルを使用する推論ワークロードでは、構造化された出力の指定もサポートされるようになりました。

Fixed timeout handling for materialized views and streaming tables
> 物化ビューおよびストリーミングテーブルの固定タイムアウトハンドリング
August 07, 2025

New timeout behavior for materialized views and streaming tables created in Databricks SQL:

  • Materialized views and streaming tables created after August 7, 2025 will have the warehouse timeout automatically applied.
  • For materialized views and streaming tables created before August 7, 2025, run CREATE OR REFRESH to synchronize the timeout setting with the timeout configuration of the warehouse.
  • All materialized views and streaming tables now have a default timeout of two days.
>

Databricks SQLで作成されたマテリアライズドビューとストリーミングテーブルの新しいタイムアウト動作:

  • 2025年8月7日以降に作成されたマテリアライズドビューとストリーミングテーブルは、ウェアハウスのタイムアウトが自動的に適用されます。
  • 2025年8月7日より前に作成されたマテリアライズドビューとストリーミングテーブルは、CREATE OR REFRESHを実行してタイムアウト設定をウェアハウスの設定と同期させる必要があります。
  • すべてのマテリアライズドビューとストリーミングテーブルの初期設定のタイムアウトは2日間です。
Serverless compute is now available in ap-southeast-3
> サーバーレス コンピューティングが ap-southeast-3 で利用可能になりました。
August 06, 2025

Serverless compute for notebooks, workflows, and Lakeflow Declarative Pipelines is now available in the Asia Pacific (Jakarta) region (ap-southeast-3). See Features with limited regional availability.

>

ノートブック、ワークフロー、Lakeflow Declarative Pipelinesのサーバーレスコンピューティングが、アジア太平洋(ジャカルタ)リージョン(ap-southeast-3)で利用可能になりました。リージョンの制限付き機能をご確認ください。

Power BI Databricks connector supports M2M OAuth
> Power BI Databricks コネクタは M2M OAuth をサポートしています。
August 05, 2025

You can now authenticate into Power BI Desktop using Machine-to-Machine (M2M) OAuth credentials. If you are currently using personal access tokens on behalf of service principals, Databricks recommends switching to the new Client credentials authentication option.

See Optional: Set up a service principal for M2M OAuth and Configure credentials in Power BI for M2M OAuth.

> サービス プリンシパルに代わってパーソナル アクセス トークンを使用している場合は、Databricks では新しいクライアント資格情報認証オプションに切り替えることをお勧めします。 オプション: M2M OAuth 用のサービス プリンシパルを設定するとM2M OAuth の Power BI で資格情報を構成するを参照してください。
OpenAI GPT OSS models now available on Mosaic AI Model Serving
> OpenAIのGPT OSSモデルはMosaic AIモデルサービングで今利用可能です。
August 05, 2025

Mosaic AI Model Serving now supports OpenAI's GPT OSS 120B and GPT OSS 20B as Databricks-hosted foundation models. You can access these models using Foundation Model APIs pay-per-token.

> Mosaic AIモデル提供は、OpenAIのGPT OSS 120BとGPT OSS 20BをDatabricksがホストするファウンデーションモデルとしてサポートしています。ファウンデーションモデルAPI従量課金を使用してこれらのモデルにアクセスできます。

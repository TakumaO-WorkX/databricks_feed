Databricks Docs Updates

Catalog Explorer navigation updates
> カタログ・エクスプローラーのナビゲーション更新
December 16, 2025

Catalog Explorer will soon receive navigation improvements to streamline workflows and help you discover and manage data assets more efficiently.

Simplified navigation:

The duplicative Catalogs tab is removed to reduce redundancy and focus on a single catalog navigation surface. DBFS and Send feedback actions move into the kebab menu for a cleaner layout.

New Suggested section:

A new Suggested tab on the Catalog Explorer landing page highlights frequently used objects, example objects for first-time users, and user favorites. This helps you quickly re-engage with important assets or discover helpful starting points.

Consolidated entry points:

Related capabilities are grouped under clearer categories to reduce visual noise and improve findability:

  • Govern – Entry point for governed tags, metastore administration, and data classification
  • Connect – Entry points for external locations, external data, credentials, and connections
  • Share – Entry points for Delta Sharing and Clean Rooms

These groupings replace scattered sub-tabs and create a more intuitive, scalable information architecture.

>

カタログ エクスプローラーには、ワークフローの効率化と、データ アセットの発見と管理をより効率的に行うための、ナビゲーションの改善が間もなく行われます。

ナビゲーションの簡素化:

重複しているカタログタブが削除されて、単一のカタログ ナビゲーション領域に集中できるようになります。DBFSとフィードバックの送信アクションは、よりすっきりとしたレイアウトのためにケバブメニューに移動されます。

新しい「おすすめ」セクション:

カタログ エクスプローラーのランディングページに新しく追加されたおすすめタブでは、頻繁に使用されるオブジェクト、初めてのユーザーのための例示的なオブジェクト、ユーザーのお気に入りなどがハイライトされます。これにより、重要なアセットをすぐに再利用したり、役立つ出発点を発見したりできます。

統合されたエントリーポイント:

関連機能が、視覚的なノイズを減らし、検索性を向上させるために、より明確なカテゴリの下にグループ化されています:

  • ガバナンス – ガバナンス タグ、メタストア管理、データ分類のエントリーポイント
  • 接続 – 外部ロケーション、外部データ、資格情報、接続のエントリーポイント
  • 共有 – Delta Sharing およびクリーンルームのエントリーポイント

これらのグループ化により、散在していたサブタブが置き換えられ、より直感的で拡張性のある情報アーキテクチャが実現します。

Confluence connector (Beta)
> コンフルエンスコネクタ(ベータ)
December 16, 2025

The fully-managed Confluence connector in Lakeflow Connect enables you to ingest Confluence spaces, pages, attachments, blogposts, labels, and classification levels into Databricks. See Configure OAuth U2M for Confluence ingestion.

> Lakeflow Connectの完全管理型Confluenceコネクタにより、Confluenceのスペース、ページ、添付ファイル、ブログ投稿、ラベル、および分類レベルをDatabricksに取り込むことができます。Confluenceへの取り込みのためのOAuth U2Mの設定を参照してください。
PostgreSQL connector in Lakeflow Connect (Public Preview)
> PostgreSQLコネクタ付きLakeflow Connect (パブリックプレビュー)
December 16, 2025

The fully-managed PostgreSQL connector in Lakeflow Connect is in Public Preview. This connector enables incremental data ingestion from PostgreSQL databases, including Amazon RDS PostgreSQL, Amazon Aurora PostgreSQL, Azure Database for PostgreSQL, Google Cloud SQL for PostgreSQL, and on-premises PostgreSQL databases. See Configure PostgreSQL for ingestion into Databricks.

> Lakeflow Connectの完全管理PostgreSQLコネクタがPublic Previewになりました。このコネクタにより、Amazon RDS PostgreSQL、Amazon Aurora PostgreSQL、Azure Database for PostgreSQL、Google Cloud SQL for PostgreSQL、およびオンプレミスのPostgreSQLデータベースからの増分データ取り込みが可能です。詳細はDatabricksへのデータ取り込みのためのPostgreSQLの設定をご覧ください。
Run read-only queries on Lakebase (Provisioned) readable secondaries from SQL editor
> Lakebaseの読み取り専用クエリを(プロビジョニング済み)読み取り専用セカンダリから SQL エディターで実行してください。
December 15, 2025

You can now connect to Lakebase (Provisioned) readable secondaries and run read-only queries from the Databricks SQL editor. See Execute read-only queries from Databricks SQL Editor and Access a database instance from the SQL editor.

>

データブリックス SQL エディターから読み取り専用クエリを実行して、Lakebase (Provisioned) の読み取り専用セカンダリにアクセスできるようになりました。「データブリックス SQL エディターから読み取り専用クエリを実行する」と「SQL エディターからデータベースインスタンスにアクセスする」をご参照ください。

Delta Sharing to external Iceberg clients is in Public Preview
> デルタ共有が外部のアイスバーグクライアントに公開プレビューになりました
December 15, 2025

You can now share tables, foreign tables, materialized views, and streaming tables to external Iceberg clients such as Snowflake, Trino, Flink, and Spark. External Iceberg clients can query shared Delta tables with zero-copy access. For details, see Enable sharing to external Iceberg clients and Iceberg clients: Read shared Delta tables.

> あなたは今、Snowflake、Trino、Flink、Sparkなどの外部Icebergクライアントとテーブル、外部テーブル、マテリアライズドビュー、ストリーミングテーブルを共有できます。外部Icebergクライアントは、ゼロコピーアクセスで共有Deltaテーブルを照会することができます。詳細については、「外部Icebergクライアントへの共有を有効にする」および「Icebergクライアント: 共有Deltaテーブルの読み取り」をご覧ください。

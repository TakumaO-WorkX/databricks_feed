Databricks Docs Updates

Dashboard enhancements and improvements
> ダッシュボードの強化と改善
February 12, 2026
  • Workspace admin theme permissions: Fixed a bug where workspace admins couldn't edit workspace-level themes.
  • Level of detail calculations: Level of detail calculations now support EXCLUDE syntax on multiple fields. See Defining expressions at a coarser level of detail
  • Color legend sorting: You can now sort color legends by another field in visualizations.
  • Pivot table cell styles: Pivot tables now support HTML, JSON, and image cell styles.
  • Pivot table cross filtering: Pivot tables now support cross filtering.
  • Table resizing: You can now resize table and pivot table headers and columns.
  • Histogram and funnel chart labels: Histogram and funnel charts can now show "total" labels.
  • Assistant for dashboard authoring improvements: Assistant for dashboard authoring now supports creating labels and setting color encoding values.
  • Multi-select parameter enhancement: Multi-select parameters now support selecting all values. See Allow multiple selections.
  • Table visualization enhancements: Table visualizations now support aggregations and measures, inherit formatting from locales, and support background color styling, including conditional formatting and color gradients. As a result of this change:
    • Table formatting now uses the visualization library's formatting capabilities, ensuring consistency and allowing inheritance from locale settings.
    • The resize columns option has been removed to reduce bugs. Set column widths in the Style tab instead.
  • Dashboard grid expansion: Dashboard layouts now support 12 columns across instead of 6, providing more flexibility for widget placement.

  • 詳細レベル計算: 詳細レベルの計算では、複数のフィールドで EXCLUDE 構文をサポートするようになりました。より粗いレベルで表現を定義するを参照してください。
  • 色凡例のソート: 視覚化で色凡例を別のフィールドでソートできるようになりました。

    • テーブルのフォーマットは、視覚化ライブラリのフォーマット機能を使用するため、一貫性が確保され、ロケール設定から継承できます。

Genie enhancements and improvements
> ジーニーの増強と改善
February 12, 2026
  • Heatmap and sankey visualizations: You can now generate heatmap and sankey visualizations in Genie.
  • Query tagging for benchmarks: In query history, Genie attaches query tags to queries generated from benchmarks, helping you differentiate workloads from end users versus benchmarks.
  • Auto-generated space titles: Space titles now automatically generate based on the context of selected tables.
  • Persistent knowledge store suggestions: Knowledge store suggestions now persist on messages so you can navigate to other tabs within Genie before returning and accepting suggestions.
  • Knowledge store suggestions on download: Genie now suggests adding query results to the knowledge store when you download them.
  • Inspect mode (Beta): Inspect mode is now in Beta. This mode automatically improves standard Genie's accuracy by reviewing the initially generated SQL, authoring smaller SQL statements to verify specific aspects of the query, and generating improved SQL as needed. See Improve response accuracy using inspect mode.

  • ヒートマップとサンキーの可視化: Genieでヒートマップとサンキーの可視化を生成できるようになりました。

  • 検査モード (ベータ版): 検査モードがベータ版で利用可能になりました。このモードでは、最初に生成されたSQLを確認し、クエリの特定の側面を検証するためのより小さなSQL文を作成し、必要に応じて改善されたSQLを生成することで、標準のGenieの精度を自動的に向上させます。検査モードを使用してレスポンスの精度を向上させるを参照してください。

File events enabled by default on new external locations
> ファイルイベントは新しい外部の場所でデフォルトで有効になります。
February 12, 2026

File events are now enabled by default when you create new external locations. The Databricks file events service sets up cloud resources to detect file changes, enabling more efficient, event-driven storage jobs and ingestion pipelines.

When you create an external location using the UI, Databricks checks for the required permissions and guides you to fix any issues or proceed with a force create. This check does not run in the API, but ingestion pipelines automatically fall back to prevent breakage.

You can disable file events for an external location after creation if needed. For more information, see Set up file events for an external location.

> ファイルイベントは、新しい外部ロケーションを作成するときに、デフォルトで有効になっています。Databricksファイルイベントサービスは、ファイルの変更を検出するためのクラウドリソースを設定し、より効率的なイベントドリブンのストレージジョブとインジェストパイプラインを可能にします。 UIを使って外部ロケーションを作成する際、Databricksは必要な権限をチェックし、問題を修正するか強制的に作成するよう案内します。このチェックはAPIでは実行されませんが、インジェストパイプラインは自動的にフォールバックして、破損を防ぎます。 必要に応じて、作成後に外部ロケーションのファイルイベントを無効化できます。詳細については、「外部ロケーションにファイルイベントをセットアップする」を参照してください。
Configure default Python package repositories for Lakeflow Spark Declarative Pipelines (Public Preview)
> Lakeflow Spark Declarative Pipelineの(パブリックプレビュー)デフォルトのPythonパッケージリポジトリを構成する
February 11, 2026

Workspace admins can now configure private or authenticated package repositories within workspaces as the default pip configuration for Lakeflow Spark Declarative Pipelines. This allows users in the workspace to install packages from internal Python repositories without explicitly defining index-url or extra-index-url values.

See Configure default Python package repositories for more details.

> ワークスペース管理者は、Lakeflow Spark Declarative Pipelinesのデフォルトのpip設定として、ワークスペース内のプライベートまたは認証済みのパッケージリポジトリを設定できるようになりました。これにより、ユーザーはindex-urlやextra-index-urlの値を明示的に定義することなく、内部のPythonリポジトリからパッケージをインストールできます。 詳細については、「Configure default Python package repositories」を参照してください。

Databricks Docs Updates

Lakehouse Federation sharing and default storage
> レイクハウス連盟の共有とデフォルトのストレージ
September 16, 2025

Delta Sharing on Lakehouse Federation is in Beta, allowing Delta Sharing data providers to share foriegn catalogs and tables. By default, data must be temporarily materialized and stored on default storage. Currently, users must manually enable the Delta Sharing for Default Storage – Expanded Access feature in the account console to use Lakehouse Federation sharing.

After Delta Sharing for Default Storage – Expanded Access is enabled by default for all Databricks users, Delta Sharing on Lakehouse Federation will automatically be available in regions where default storage is supported.

See Default storage in serverless workspaces and Add foreign schemas or tables to a share.

> レイクハウスフェデレーションでのデルタシェアリングはベータ版で、デルタシェアリングのデータプロバイダーが外国のカタログとテーブルを共有できるようになっています。デフォルトでは、データを一時的に具体化して、デフォルトのストレージに保存する必要があります。現在、ユーザーは「アカウントコンソールでデフォルトストレージのデルタシェアリング - 拡張アクセス」機能を手動で有効にする必要があります。 デフォルトストレージのデルタシェアリング - 拡張アクセスが全てのDatabricksユーザーにデフォルトで有効になると、デフォルトのストレージがサポートされているリージョンでレイクハウスフェデレーションのデルタシェアリングが自動的に利用可能になります。 「サーバーレスワークスペースのデフォルトストレージ」と「共有に外国のスキーマやテーブルを追加する」をご覧ください。

Delta Sharing on Lakehouse Federation is in Beta
> デルタ共有はレイクハウス連合ベータ版にあります。
September 16, 2025

You can now use Delta Sharing to share foreign schemas and tables created with query federation in Databricks-to-Databricks sharing and open sharing. See Add foreign schemas or tables to a share and Read data in a shared foreign table or foreign schema.

> データブリックス間の共有とオープン共有においてクエリフェデレーションを使用して作成された外部スキーマとテーブルを共有するために、Delta Sharing を使用できるようになりました。詳細については、「Add foreign schemas or tables to a share」と「Read data in a shared foreign table or foreign schema」をご覧ください。
Reload notification in workspaces
> ワークスペースでのリロード通知
September 13, 2025

In an upcoming release, a message to reload your workspace tab will display if your workspace tab has been open for a long time without refreshing. This will help ensure you are always using the latest version of Databricks with the newest features and fixes.

>

今後のリリースでは、ワークスペースのタブを長時間更新せずに開いていた場合、ワークスペースのタブを再読み込みするメッセージが表示されるようになります。これにより、最新の機能やバグ修正を含むDatabricksの最新バージョンを常に使用することができます。

Mount Delta shares to an existing shared catalog
> デルタ山の株式は既存の共有カタログに共有されます。
September 12, 2025

Delta Sharing recipients can now mount shares received from their Delta Sharing provider to an existing shared catalog. Previously, recipients needed to create a new catalog for each new share. See Create a catalog from a share.

> デルタ共有の受信者は、デルタ共有プロバイダーから受け取った共有を既存の共有カタログにマウントできるようになりました。以前は、受信者が新しい共有ごとに新しいカタログを作成する必要がありました。共有からカタログを作成するをご覧ください。

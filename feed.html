Databricks Docs Updates

Dashboard enhancements and improvements
> ダッシュボードの機能強化と改善
February 26, 2026
  • Unity Catalog metric view lineage: Dashboard info now shows Unity Catalog metric views in the lineage section.
  • Dashboard refresh cancellation: Cancelling a dashboard refresh now shows the last available data instead of an error.
  • DD-MM-YYYY date format: The DD-MM-YYYY date format is now available for date fields.
  • Date range parameter "All" selection: Date range parameters now support All. When All is selected, the parameter value is null and all rows are returned. See Apply date range parameters.
  • Genie agentic experience (Public Preview): Genie for dashboard insights now offers an agentic experience (Public Preview) by default.
  • Point map recentering: Point maps now recenter upon relevant points when a filter is applied.
  • Datetime drop-down menu fix: Fixed an issue where datetime drop-down menus could not be set in the global filters panel.
>
  • Unity Catalogメトリックビューラインテージ: ダッシュボード情報に、Unity Catalogメトリックビューがラインテージセクションに表示されるようになりました。
  • ダッシュボードリフレッシュのキャンセル: ダッシュボードのリフレッシュをキャンセルすると、エラーではなく最新の利用可能なデータが表示されるようになりました。
  • DD-MM-YYYY日付フォーマット: DD-MM-YYYY日付フォーマットが日付フィールドで利用可能になりました。
  • 日付範囲パラメータの「すべて」の選択: 日付範囲パラメータで「すべて」がサポートされるようになりました。「すべて」が選択された場合、パラメータ値はnullとなり、すべての行が返されます。日付範囲パラメータの適用を参照してください。
  • Genieのエージェンシー体験(パブリックプレビュー): ダッシュボードインサイトのGenieがデフォルトでエージェンシー体験(パブリックプレビュー)を提供するようになりました。
  • ポイントマップの再センタリング: フィルターが適用されると、ポイントマップが関連するポイントを中心に再センタリングされるようになりました。
  • 日時ドロップダウンメニューの修正: グローバルフィルターパネルで日時ドロップダウンメニューを設定できない問題が修正されました。
Genie enhancements and improvements
> ジーニーの強化と改良
February 26, 2026
  • Metric view display names: Genie now uses metric view display names and display formats for query result tables.
  • Automatic JOIN relationships: Creating and updating Genie spaces using the API now automatically creates JOIN relationships based on the primary and foreign keys defined in source tables.
  • Context tab for space consumers: A new Context tab is now available for Genie space consumers with CAN VIEW or CAN RUN permissions. Consumers can view the space description and common questions at any point during their Genie interactions. See Use a Genie space to explore business data.
  • Agent mode (Beta) thinking traces: Agent mode now streams thinking traces in-line for improved readability.
  • Agent mode (Beta) parallel queries: Agent mode now runs queries in parallel for faster results.
>
  • メトリックビューの表示名: Genieは、クエリ結果テーブルのメトリックビューの表示名と表示形式を使用しています。
  • 自動JOIN関係: APIを使用してGenieスペースを作成および更新すると、ソーステーブルに定義されたプライマリーキーと外部キーに基づいて自動的にJOIN関係が作成されます。
  • スペースコンシューマー向けのコンテキストタブ: CAN VIEWまたはCAN RUN権限を持つGenieスペースコンシューマーに、新しいコンテキストタブが利用できるようになりました。コンシューマーは、Genieインタラクション中いつでもスペースの説明と一般的な質問を表示できます。 Genieスペースを使用してビジネスデータを調査するをご覧ください。
  • エージェントモード(ベータ)の思考履歴: エージェントモードでは、読みやすさを向上させるために、思考履歴がインラインでストリーミングされるようになりました。
  • エージェントモード(ベータ)の並列クエリ: エージェントモードでは、より高速な結果を得るために、クエリが並列で実行されるようになりました。
HITRUST compliance controls (Public Preview)
> HITRUST コンプライアンス コントロール(パブリック プレビュー)
February 26, 2026

You can now enable HITRUST compliance controls to manage risk and demonstrate security and privacy compliance. See HITRUST.

>

HITRUSTコンプライアンス管理をオンにして、リスクを管理し、セキュリティとプライバシーのコンプライアンスを実証できるようになりました。HITRUSTを参照してください。

Python unit testing in the workspace
> Pythonのユニットテストワークスペース内
February 26, 2026

Databricks now provides integrated Python unit testing tools in the workspace. Use the tests sidebar, inline execution glyphs, and testing results bottom panel tab to discover, run, and debug pytest-based tests. See Python unit testing in the workspace.

>

Databricksはワークスペースに統合されたPythonユニットテストツールを提供するようになりました。テストサイドバー、インライン実行グリフ、およびテスト結果のボトムパネルタブを使用して、pytest ベースのテストを検出、実行、およびデバッグできます。ワークスペースでのPythonユニットテストをご覧ください。

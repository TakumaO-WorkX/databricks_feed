Databricks Docs Updates

View and restore your recent workspace tab sessions
> 最近のワークスペースタブセッションを表示して復元する
January 30, 2026

You can now view your past workspace tab sessions and restore them. See Switch between authoring contexts.

>

これで過去のワークスペースタブセッションを表示し、それらを復元することができます。編集コンテキストの切り替えをご覧ください。

Paste images into notebooks
> イメージをノートブックに貼り付ける
January 30, 2026

You can now copy images from your local file system and paste them into markdown cells in Databricks notebooks using ⌘ + V (mac) or Ctrl + V (Windows). See Display images.

>

ローカルファイルシステムからイメージをコピーし、Databricksノートブックのマークダウンセルに貼り付けることができるようになりました。⌘ + V (mac) または Ctrl + V (Windows) を使用してください。イメージの表示を参照してください。

Dashboard visualization and data improvements
> ダッシュボードの視覚化とデータの改善
January 29, 2026
  • Fixed access request availability: Resolved an issue where the option to request access was not available and a "Resource not found" error was incorrectly shown.
  • Full-width pivot tables: Widgets now render pivot tables at full width for better readability.
  • Tooltip encodings on charts: Combo and dual-axis charts now support tooltip encodings.
  • Image rendering in pivot tables: Cells can now display images directly in pivot tables.
  • Dark mode scrollbar visibility: Scrollbars are now easier to see when using dark mode.
  • Increased pivot data download limits: Download limits now support up to 10,000 rows by 1,000 columns for pivoted data.
  • Fixed filter paste behavior: Resolved an issue where pasting a comma-separated list into a single-select filter only parsed the first element instead of keeping all values.
  • Fixed stacked chart label rendering: Labels on stacked charts with a quantitative x-axis now render correctly in notebooks and in the SQL Editor.
>
  • 固定アクセス要求の利用可能性：アクセスを要求するオプションが利用できず、「リソースが見つかりません」というエラーが誤って表示されていた問題を解決しました。
  • 全幅のピボットテーブル：ウィジェットがピボットテーブルを全幅で表示するようになり、よりわかりやすくなりました。
  • チャートのツールチップエンコーディング：コンボチャートと二重軸チャートでツールチップエンコーディングがサポートされるようになりました。
  • ピボットテーブルでの画像の表示：ピボットテーブルの中で直接、画像を表示できるようになりました。
  • ダークモードでのスクロールバーの表示：ダークモードでも、スクロールバーがよりはっきりと表示されるようになりました。
  • ピボットデータのダウンロード制限の増加：ダウンロード制限が、最大10,000行×1,000列のピボットデータに対応するようになりました。
  • フィルターへの貼り付けの修正：単一選択フィルターにカンマ区切りのリストを貼り付けると、最初の要素しか解析されない問題を解決しました。
  • 積み上げチャートのラベルの表示の修正：数量軸を持つ積み上げチャートのラベルが、ノートブックとSQL Editorで正しく表示されるようになりました。
Genie API and user experience improvements
> ジーニーAPIとユーザー体験の改善
January 29, 2026
  • Full query result downloads: A new API endpoint supports downloading complete result sets up to about 1 GB.
  • Fixed chart generation from prompts: Resolved an issue where requesting charts directly in the prompt would not create visualizations.
  • Collapsible error messages: Errors now appear in a collapsible modal for improved readability.
  • Catalog permission warnings: Users now see an informative banner when they lack USE CATALOG permissions on attached tables, rather than encountering errors at SQL execution time.
>
  • クエリ結果のフルダウンロード: 新しいAPIエンドポイントでは、約1GBまでの完全なクエリ結果セットをダウンロードできるようになりました。
  • プロンプトからのチャート生成の修正: プロンプトで直接チャートを要求する際に、可視化が作成されない問題を修正しました。
  • 折りたたみ式のエラーメッセージ: エラーがより読みやすい折りたたみ式のモーダルで表示されるようになりました。
  • カタログアクセス権の警告: ユーザーは、SQL実行時にエラーが発生するのではなく、接続されたテーブルに対するUSE CATALOG権限がない場合に、それを示す情報バナーが表示されるようになりました。
Serverless workspaces are now generally available
> サーバーレスワークスペースが一般提供されるようになりました。
January 29, 2026

Serverless workspaces are now generally available. Serverless workspaces come pre-configured with serverless compute and default storage, providing a fully-managed, enterprise-ready SaaS experience. See Create a serverless workspace.

> サーバーレスワークスペースが一般提供されるようになりました。サーバーレスワークスペースはサーバーレスコンピューティングと既定のストレージが事前構成されており、完全に管理された企業向けSaaSの体験を提供します。サーバーレスワークスペースの作成を参照してください。

Databricks Docs Updates

Google Gemini 3.1 Flash Lite now available as a Databricks-hosted model
> Googleジェミニ3.1 Flash Liteがデータブリックスホストのモデルとして利用可能になりました。
March 06, 2026

Mosaic AI Model Serving now supports Google Gemini 3.1 Flash Lite as a Databricks-hosted model.

To access this model, use:

  • Foundation Model APIs pay-per-token
    • Query vision models
  • Batch inference workloads using AI Functions
> モザイクAIモデルサービングは、DatabricksホストのモデルとしてGoogle Gemini 3.1 Flash Liteをサポートするようになりました。 このモデルにアクセスするには、以下のいずれかを使用します: - トークン課金制の基盤モデルAPI - ビジョンモデルのクエリ - AIファンクションを使ったバッチ推論ワークロード
Endpoint telemetry for custom model serving endpoints (Beta)
> エンドポイントテレメトリカスタムモデルサービングエンドポイント(ベータ版)
March 06, 2026

You can now configure endpoint telemetry to persist logs, traces, and metrics from custom model serving endpoints into Unity Catalog Delta tables using OpenTelemetry. This enables root cause analysis, application health monitoring, and compliance retention using standard SQL queries.

See Persist custom model serving data to Unity Catalog.

>

ユニティカタログデルタテーブルにOpenTelemetryを使用して、カスタムモデルサービングエンドポイントからログ、トレース、メトリックを永続化するようにエンドポイントテレメトリを構成できるようになりました。これにより、標準SQLクエリを使用して、根本原因分析、アプリケーションの健全性監視、コンプライアンス保持が可能になります。

「ユニティカタログへのカスタムモデルサービングデータの永続化」を参照してください。

Databricks Apps telemetry (Beta)
> データブリックス・アプリテレメトリー（ベータ版）
March 06, 2026

You can now configure telemetry for your Databricks apps to collect traces, logs, and metrics and persist them to Unity Catalog tables using the OpenTelemetry (OTel) protocol. After you enable app telemetry, Databricks automatically captures system logs and usage events.

See Configure telemetry for Databricks Apps.

> あなたは今、OpenTelemetry (OTel)プロトコルを使ってトレース、ログ、メトリクスを収集し、Unity Catalogのテーブルに保存するよう、Databricksアプリのテレメトリーを設定できます。アプリのテレメトリーを有効にすると、Databricksは自動的にシステムログと使用イベントをキャプチャします。 Databricksアプリのテレメトリーを設定するをご覧ください。
OpenAI GPT-5.4 now available as a Databricks-hosted model
> オープンAI GPT-5.4がデータブリックスのホストされたモデルとして利用可能になりました。
March 05, 2026

Mosaic AI Model Serving now supports OpenAI GPT-5.4 as a Databricks-hosted model. You can access this model using Foundation Model APIs pay-per-token.

Customers are responsible for ensuring their compliance with the terms of OpenAI's Acceptable Use Policy.

> モザイクAIモデルサーバーがOpenAI GPT-5.4をデータブリックスホストのモデルとしてサポートするようになりました。Foundation Model APIsのトークン課金を使ってこのモデルにアクセスできます。 顧客は、OpenAIの利用規約に従っていることを確認する責任があります。
Tree view of the workspace browser side panel (Public Preview)
> ワークスペース ブラウザー側面のツリービュー (パブリック プレビュー)
March 05, 2026

The workspace browser side panel now includes a tree view that provides a flexible, tree-based view of your workspace folder hierarchy. The tree view allows you to expand descendant folders, change the root of the tree, and navigate your workspace while keeping the tree steady. This improves the multi-asset authoring experience by providing better control over workspace navigation.

See Tree view of the workspace browser side panel (Public Preview).

> ワークスペース・ブラウザのサイドパネルには、ワークスペース・フォルダ階層の柔軟な、ツリー型ビューを提供するツリービューが含まれるようになりました。ツリービューを使えば、子フォルダを展開したり、ツリーのルートを変更したり、ワークスペースを移動したりできます。これにより、ワークスペース・ナビゲーションを better control することができるようになり、マルチアセットの制作体験が向上します。 ワークスペース・ブラウザのサイドパネルのツリービュー (パブリック・プレビュー)をご覧ください。

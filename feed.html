Databricks Docs Updates

AI/BI dashboard enhancements and global filters improvements
> AIおよびBI管理ダッシュボードの機能強化と、グローバルフィルターの改善
October 02, 2025
  • Fixed global filters panel overflow: Resolved an issue where the global filters panel extended beyond its intended width.
  • Access for account-level users: Users without a workspace can now access dashboards shared with them that were published without embedded credentials. See What are embedded credentials?.
  • Preview fields and measures for metric views: Authors can now preview available fields and measures for metric views in the Add data dialog.
  • Query insights in dashboard dataset editor: Query insights are now available in the dashboard dataset editor to help analyze query performance.
  • Copy and paste widgets and pages: Dashboard widgets and pages can now be copied and pasted between dashboards.
  • Top or bottom N categories in bar charts: Bar charts can now display the top or bottom N categories. To edit, open the kebab menu for the categorical axis dimension and set the number under Default number of categories.
>
  • 固定グローバルフィルターパネルのオーバーフロー: グローバルフィルターパネルが意図した幅を超えて拡張されていた問題を解決しました。
  • アカウントレベルのユーザーのアクセス: ワークスペースを持たないユーザーが、埋め込み資格情報なしで公開されたダッシュボードを共有して閲覧できるようになりました。埋め込み資格情報とは何ですか?をご覧ください。
  • メトリックビューのフィールドとメジャーのプレビュー: 作成者はデータの追加ダイアログでメトリックビューの利用可能なフィールドとメジャーをプレビューできるようになりました。
  • ダッシュボードデータセットエディターでのクエリインサイト: クエリパフォーマンスの分析に役立つクエリインサイトがダッシュボードデータセットエディターで利用できるようになりました。
  • ウィジェットとページのコピーアンドペースト: ダッシュボードのウィジェットとページをダッシュボード間でコピーアンドペーストできるようになりました。
  • 棒グラフでのトップまたはボトムN個のカテゴリ: 棒グラフでは、トップまたはボトムのN個のカテゴリを表示できるようになりました。編集するには、カテゴリ軸の次元のケバブメニューを開き、デフォルトのカテゴリ数の下の数値を設定してください。

> ジーニー空間の拡張と思考トレース機能

Semantic metadata in metric views
> セマンティックメタデータにメトリックビューを使用する
October 02, 2025

You can now define semantic metadata in a metric view. Semantic metadata helps AI tools such as Genie spaces and AI/BI dashboards interpret and use your data more effectively.

To use semantic metadata, your metric view must use YAML specification version 1.1 or higher and run on DBR 17.2 or above. The corresponding Databricks SQL version is 2025.30, available in the Preview channel for SQL warehouses.

See Use semantic metadata in metric views and Upgrade considerations.

> メトリックビューでセマンティックメタデータを定義できるようになりました。セマンティックメタデータにより、Genie spacesやAI/BIダッシュボードなどのAIツールがデータを効果的に解釈し活用できるようになります。 セマンティックメタデータを使用するには、メトリックビューがYAML仕様バージョン1.1以降を使用し、DBR 17.2以降で実行されている必要があります。対応するDatabricks SQLのバージョンは2025.30で、SQLウェアハウスのプレビューチャンネルで利用可能です。 「メトリックビューでセマンティックメタデータを使用する」と「アップグレードの考慮事項」を参照してください。
Convert to Unity Catalog managed table from external table
> Unity Catalog管理テーブルへの外部テーブルの変換
October 02, 2025

The ALTER TABLE ... SET MANAGED command is now generally available. This command seamlessly converts Unity Catalog external tables to managed tables. It allows you to take full advantage of Unity Catalog managed table features, such as enhanced governance, reliability, and performance. See Convert an external table to a managed Unity Catalog table.

>

ALTER TABLE ... SET MANAGEDコマンドが一般的に利用可能になりました。このコマンドは、Unity Catalogの外部テーブルを管理テーブルに自然に変換します。これにより、ガバナンス、信頼性、パフォーマンスなどの管理テーブルの機能を十分に活用できます。外部テーブルをUnity Catalog管理テーブルに変換するをご覧ください。

New permissions for the Databricks GitHub App
> データブリックスGitHubアプリの新しい権限
October 01, 2025

If you own a Databricks account with the Databricks GitHub app installed, you may receive an email titled "Databricks is requesting updated permissions" from GitHub.

This is a legitimate request from Databricks. It asks you to approve a new permission that allows Databricks to read your GitHub account email(s). Granting this permission will let Databricks retrieve and save your primary GitHub account email to your Linked Git credential in Databricks. In an upcoming feature, this will ensure that commits made from Databricks are properly linked to your GitHub identity.

If you don't accept the new permission, your Linked Git credential will still authenticate with GitHub. However, future commits from this credential will not be associated with your GitHub account identity

> Databricksアカウントにデータブリックスの[GitHub app](https://github.com/apps/databricks)がインストールされている場合、GitHub から「Databricksは更新された権限を要求しています」というタイトルのメールが届く可能性があります。 これはデータブリックスからの正当な要求です。Databricksが[GitHub アカウントのメール](https://docs.github.com/en/rest/users/emails)を読み取ることを許可するよう求めています。この権限を許可すると、データブリックスはお客様の主要なGitHubアカウントのメールアドレスを取得し、Databricksの紐付けられたGitの資格情報に保存できるようになります。今後の機能では、これによりDatabricksからのコミットが[GitHub アカウントに適切に紐付けられるようになります](https://docs.github.com/en/account-and-profile/how-tos/setting-up-and-managing-your-personal-account-on-github/managing-email-preferences/setting-your-commit-email-address)。 新しい権限を承認しない場合でも、紐付けられたGitの資格情報はGitHubで認証できます。ただし、この資格情報から行う今後のコミットは[GitHub アカウントのアイデンティティに関連付けられません](https://docs.github.com/en/account-and-profile/how-tos/setting-up-and-managing-your-personal-account-on-github/managing-email-preferences/setting-your-commit-email-address)。

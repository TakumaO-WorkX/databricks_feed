Declarative Feature Engineering APIs (Beta)
> 宣言的な特徴エンジニアリング API (ベータ版)
March 09, 2026

Declarative Feature Engineering APIs are now available in Beta in the AWS us-east-1 and us-west-2 regions. These APIs let you declaratively define and compute time-windowed aggregation features from your data sources.

With declarative feature engineering, you can:

  • Define features in Unity Catalog.
  • Materialize batch features to Delta tables or Lakebase.
  • Serve Unity Catalog models using online model serving.

See Declarative feature engineering and managed pipelines and Materialize and serve declarative features.

> デクララティブ特徴エンジニアリングAPIが、AWS us-east-1およびus-west-2リージョンでベータ版として利用可能になりました。これらのAPIを使用すると、データソースから時間枠付きの集計特徴を宣言的に定義し、計算することができます。 デクララティブ特徴エンジニアリングによって以下のことができます: - Unity Catalogで特徴を定義する - バッチ特徴をDeltaテーブルまたはLakebaseにマテリアライズする - Unity Catalogモデルをオンラインモデル提供で提供する 詳細は、『デクララティブ特徴エンジニアリングと管理パイプライン』と『デクララティブ特徴のマテリアライズと提供』をご覧ください。
BI compatibility mode for metric views in Power BI (Beta)
> ビジネスインテリジェンス互換モードのメトリックビューをPower BI（ベータ）で使用する
March 09, 2026

BI compatibility mode for metric views in Power BI is now available in Beta. You can query Databricks metric views in Power BI using standard SQL aggregation functions by enabling BI compatibility mode when creating a connection. It automatically rewrites aggregate functions to use the correct measure definitions. See Query metric views in Power BI.

> Power BI のメトリックビューでBI互換モードが現在ベータ版で利用可能になりました。BI互換モードを有効にすることで、標準SQLの集計関数を使用してPower BIからDatabricksのメトリックビューにクエリを実行できます。集計関数は自動的に正しいメジャー定義に書き換えられます。詳細はPower BIでメトリックビューを照会するをご覧ください。

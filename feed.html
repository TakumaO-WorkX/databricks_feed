Fixed timeout handling for materialized views and streaming tables
> [翻訳エラー: 404 models/gemini-pro is not found for API version v1beta, or is not supported for generateContent. Call ListModels to see the list of available models and their supported methods.]
August 07, 2025

New timeout behavior for materialized views and streaming tables created in Databricks SQL:

  • Materialized views and streaming tables created after August 7, 2025 will have the warehouse timeout automatically applied.
  • For materialized views and streaming tables created before August 7, 2025, run CREATE OR REFRESH to synchronize the timeout setting with the timeout configuration of the warehouse.
  • All materialized views and streaming tables now have a default timeout of two days.

Databricks Docs Updates

Dashboard filter and counter enhancements
> ダッシュボードのフィルターとカウンターの機能強化
November 06, 2025
  • Per-widget filter enhancements: Per-widget filters now support date ranges, relative date ranges, and number filters.
  • Counter visualization improvements: You can now use counters to track and compare metrics over time and show sparklines. To create a counter with a sparkline, select a main value and a date column. See Counter visualization.
  • Metric view metadata support: AI/BI dashboards now consume Unity Catalog metadata from metric views, including formatting and display name.
  • Horizontal chart scrollbars on hover: Scrollbars for horizontal charts now show on hover.
  • Fixed custom calculations display in draft mode: Resolved an issue where visualizations using custom calculations were not displaying in draft mode.
  • Fixed Slack notification OAuth token field: Resolved a bug where the OAuth token field appeared pre-filled with placeholder text despite no prior token being provided for dashboard subscription Slack notifications.
>
  • ウィジェットごとのフィルター機能の強化: ウィジェットごとのフィルターでは、日付範囲、相対的な日付範囲、数値フィルターをサポートするようになりました。
  • カウンターの視覚化の改善: カウンターを使って時間の経過とともにメトリクスを追跡・比較し、スパークラインを表示することができるようになりました。スパークラインつきのカウンターを作成するには、メイン値と日付列を選択してください。カウンターの視覚化をご確認ください。
  • メトリクスビューのメタデータサポート: AI/BIダッシュボードでは、Unity Catalogのメタデータ(フォーマットや表示名)をメトリクスビューから取得できるようになりました。
  • ホバー時の水平チャートスクロールバー: 水平チャートのスクロールバーがホバー時に表示されるようになりました。
  • ドラフトモードでの正常な独自計算の表示: 独自計算を使用する視覚化がドラフトモードで正しく表示されるようになりました。
  • Slackの通知OAuthトークンフィールドの修正: ダッシュボードのSubscription Slack通知で、OAuthトークンフィールドがダミーテキストで事前入力されていた問題を解決しました。
Genie visualization and query result improvements
> ジーニー視覚化とクエリ結果の改善
November 06, 2025
  • Text explanations for empty results: Genie now includes text explanations of query results when query results are empty.
  • Improved auto-generated charts: Visualizations now use query result metadata for higher-quality auto-generated charts.
  • Improved date formatting: The process for formatting date results in visualizations has been improved.
>
  • クエリ結果が空の場合のテキスト説明: Genieは、クエリ結果が空の場合にクエリ結果のテキスト説明を含むようになりました。
  • 自動生成チャートの改善: ビジュアライゼーションでは、クエリ結果のメタデータを使って高品質な自動生成チャートを使うようになりました。
  • 日付の書式設定の改善: ビジュアライゼーションの日付結果の書式設定プロセスが改善されています。

List MCP servers in Databricks Marketplace
> MCP サーバーを Databricks Marketplace にリストアップする
November 06, 2025

Databricks Marketplace now supports listing Model Context Protocol (MCP) servers. Providers can list MCP servers to distribute AI tools and other Databricks users can install MCP servers from Marketplace to connect AI agents to external data sources, SaaS tools, and developer APIs.

See List your data product in Databricks Marketplace and Get access to external MCP servers.

> データブリックスマーケットプレイスはNow、Model Context Protocol（MCP）サーバーの掲載をサポートしています。プロバイダーはMCPサーバーを掲載して、AI ツールを配布することができ、DatabricksユーザーはマーケットプレイスからMCPサーバーをインストールして、外部データソース、SaaSツール、開発者APIにAIエージェントを接続することができます。 データブリックスマーケットプレイスであなたのデータ製品を掲載すると外部MCPサーバーにアクセスするを参照してください。
Genie Research agent mode can soon use models served through Amazon Bedrock
> ジーニー・リサーチのエージェントモードは、Amazon Bedrock経由でサービスされたモデルを使用できるようになる予定です。
November 05, 2025

Genie Research agent mode will soon be able to use models served through Amazon Bedrock when partner-powered AI features are enabled.

>

ジーニー・リサーチのエージェントモードは、パートナー主導のAI機能が有効になると、Amazon Bedrock を介して提供されるモデルを使用できるようになる予定です。

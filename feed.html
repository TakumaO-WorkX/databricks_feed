Databricks Docs Updates

Some partner-powered AI features will soon be available in-country for Canada, Brazil, Japan, Korea, and the UK
> カナダ、ブラジル、日本、韓国、英国の一部のパートナー駆動型AI機能が近日中にその国で利用可能となる予定です。
February 24, 2026

The following features will soon be available through models hosted in-country in Brazil, Canada, Japan, Korea, and the UK:

  • Assistant chat and cell actions
  • Genie spaces (excluding research agent)
  • AI generated comments
  • AI/BI dashboards

Workspaces in Brazil, Canada, Japan, Korea, and the UK will soon be able to use these features with Enforce data processing within workspace Geography for Designated Services enabled and partner-powered AI features enabled. For these features in the affected countries, in-country data routing will be enforced, meaning data will not leave the country.

If partner-powered AI features is disabled, the Assistant can use Databricks-hosted models only if Enforce data processing within workspace Geography for Designated Services is disabled.

All other designated services continue to comply with data residency managed by Databricks Geos. Some AI capabilities that depend on models not currently hosted in these countries might no longer be available when Enforce data processing within workspace Geography for Designated Services is enabled. See Databricks Geos: Data residency for more information on which features require cross-geo processing.

>

次の機能は、ブラジル、カナダ、日本、韓国、英国に設置されたモデルを使って間もなく利用可能になります:

  • アシスタントチャットとセルアクション
  • Genieスペース(リサーチエージェントを除く)
  • AI生成コメント
  • AI/BIダッシュボード

ブラジル、カナダ、日本、韓国、英国のワークスペースでは、指定サービスのためのワークスペース地理内でのデータ処理の強制が有効になり、パートナー提供のAI機能が有効になれば、これらの機能を使えるようになります。これらの国での機能では、国内データルーティングが強制され、データが国外に出ることはありません。

パートナー提供のAI機能が無効化されている場合、アシスタントはDatabricksがホストするモデルのみを使用できますが、その場合は指定サービスのためのワークスペース地理内でのデータ処理の強制も無効化する必要があります。

その他の指定サービスは、Databricks Geosが管理するデータ保管場所の要件を引き続き遵守します。指定サービスのためのワークスペース地理内でのデータ処理の強制を有効にした場合、これらの国にホストされていないモデルに依存する一部のAI機能は利用できなくなる可能性があります。クロスジオ処理を必要とする機能の詳細については、Databricks Geos: データ保管場所を参照してください。

Personal access tokens preserved when CAN USE permission is revoked
> 個人アクセストークンは、CAN USE権限が取り消されても保持されます。
February 23, 2026

When you revoke a user's CAN USE permission, their personal access tokens become unusable but are not deleted. If the permission is restored, the same tokens become active again. See Manage personal access token permissions.

> ユーザーのCAN USE許可を取り消すと、ユーザーの個人アクセストークンは使用できなくなりますが、削除されません。権限が復元されると、同じトークンが再び有効になります。個人アクセストークンの権限の管理を参照してください。

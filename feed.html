Databricks Docs Updates

Anthropic Claude Sonnet 4.5 now available as a Databricks-hosted model
> アンソロピック・クロード・ソネット 4.5 がデータブリックスでホストされたモデルとして利用可能になりました。
October 03, 2025

Mosaic AI Model Serving now supports Anthropic Claude Sonnet 4.5 as a Databricks-hosted model. You can access this model using Foundation Model APIs pay-per-token.

>

モザイクAIモデルサービングは、Databricksホストのモデルとして、Claude Sonnet 4.5をサポートしています。Foundation Model APIs pay-per-tokenを使ってこのモデルにアクセスできます。

Notebook improvements
> ノートブックの改善
October 03, 2025

The following notebook improvements are now available:

  • The cell execution minimap now appears in the right margin of notebooks. Use the minimap to get a visual overview of your notebook's run status and quickly navigate between cells. See Cell execution minimap.
  • Use Databricks Assistant to help diagnose and fix environment errors, including library installation errors. See Debug environment errors.
  • When reconnecting to serverless notebooks, sessions are automatically restored with the notebook's Python variables and Spark state. See Automated session restoration for serverless notebooks.
  • Pyspark authoring completion now supports agg, withColumns, withColumnsRenamed, and filter/where clauses. See Get inline code suggestions: Python and SQL examples.
  • Databricks now supports importing and exporting IPYNB notebooks up to 100 MB. Revision snapshot autosaving, manual saving, and cloning are supported for all notebooks up to 100 MB. See Notebook sizing.
  • When cloning and exporting notebooks, you can now choose whether to include cell outputs or not. See Manage notebook format.
>

以下のノートブック機能の改善が利用可能になりました:

  • セル実行ミニマップがノートブックの右マージンに表示されるようになりました。ミニマップを使って、ノートブックの実行状況を視覚的に把握し、セル間を素早く移動できます。セル実行ミニマップをご覧ください。
  • Databricks Assistantを使って、ライブラリのインストールエラーなど、環境エラーの診断と修正ができるようになりました。環境エラーのデバッグをご覧ください。
  • サーバーレスノートブックに再接続する際、ノートブックのPython変数とSpark状態が自動的に復元されるようになりました。サーバーレスノートブックの自動セッション復元をご覧ください。
  • Pyspark作成時の自動補完に、agg、withColumns、withColumnsRenamed、filter/where句が追加されました。インラインコード提案を取得: Python およびSQL の例をご覧ください。
  • Databricksでは、最大100MBのIPYNBノートブックのインポートとエクスポートがサポートされるようになりました。リビジョンスナップショットの自動保存、手動保存、クローンも100MBまでのすべてのノートブックで利用可能です。ノートブックのサイズをご覧ください。
  • ノートブックのクローンとエクスポート時に、セル出力の含めるかどうかを選択できるようになりました。ノートブック形式の管理をご覧ください。

AI/BI dashboard enhancements and global filters improvements
> AIおよびBIダッシュボードの機能強化とグローバルフィルターの改善
October 02, 2025
  • Fixed global filters panel overflow: Resolved an issue where the global filters panel extended beyond its intended width.
  • Access for account-level users: Users without a workspace can now access dashboards shared with them that were published without embedded credentials. See What are embedded credentials?.
  • Preview fields and measures for metric views: Authors can now preview available fields and measures for metric views in the Add data dialog.
  • Query insights in dashboard dataset editor: Query insights are now available in the dashboard dataset editor to help analyze query performance.
  • Copy and paste widgets and pages: Dashboard widgets and pages can now be copied and pasted between dashboards.
  • Top or bottom N categories in bar charts: Bar charts can now display the top or bottom N categories. To edit, open the kebab menu for the categorical axis dimension and set the number under Default number of categories.
>
  • グローバルフィルターパネルのオーバーフローを修正: グローバルフィルターパネルが意図した幅を超えて広がる問題を解決しました。
  • アカウントレベルユーザーのアクセス: ワークスペースを持たないユーザーも、埋め込み資格情報なしで公開されたダッシュボードを共有して閲覧できるようになりました。 埋め込み資格情報とは何ですか?をご覧ください。
  • メトリックビューのフィールドとメジャーのプレビュー: 作成者はデータの追加ダイアログでメトリックビューの使用可能なフィールドとメジャーをプレビューできるようになりました。
  • ダッシュボードデータセットエディターでのクエリ分析: クエリのパフォーマンス分析のために、ダッシュボードデータセットエディターでクエリ分析が利用可能になりました。
  • ウィジェットとページのコピーアンドペースト: ダッシュボードのウィジェットとページを他のダッシュボードにコピーアンドペーストできるようになりました。
  • 棒グラフの上位または下位Nカテゴリ: 棒グラフで上位または下位のNカテゴリを表示できるようになりました。編集するには、カテゴリ軸の次元のケバブメニューを開き、デフォルトのカテゴリ数の下の数値を設定してください。
Genie space enhancements and thinking trace features
> ジーニー宇宙強化機能と思考追跡機能
October 02, 2025
  • Add tables and metric views to spaces: Users can now add tables and metric views to a space and analyze them together.
  • Benchmarks scoring improvement: Benchmarks now treat extra rows or columns in Genie's output as incorrect. See Interpret ratings.
  • Thinking trace for responses: Each Genie response now includes a thinking trace that shows how the prompt was interpreted, along with the tables and example SQL statements that were used. See Review a response.
>
  • スペースにテーブルとメトリックビューを追加: ユーザーは、スペースにテーブルとメトリックビューを追加し、それらを一緒に分析できるようになりました。
  • ベンチマークのスコアリングの改善: ベンチマークでは、Genieの出力に余分な行や列がある場合、それを正しくないものとして扱うようになりました。レーティングの解釈をご覧ください。
  • レスポンスの思考過程: Genieのレスポンスには、プロンプトがどのように解釈されたか、およびどのテーブルと例のSQL文が使用されたかを示す思考過程が含まれるようになりました。レスポンスを確認するをご覧ください。

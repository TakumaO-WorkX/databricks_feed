Databricks Docs Updates

Databricks One search and discovery improvements
> データブリックスのワンサーチと探索の改善
January 22, 2026
  • Databricks Apps filtering: Databricks One consumers can now filter to include or exclude unavailable Databricks apps when browsing. Unavailable apps are those with a status other than Running, such as Stopped, Deploying, or Crashed. See App status.
  • Dashboard widget title search: Search now supports dashboard widget titles.
  • Enhanced search rankings: Certified and favorite content is now boosted in search rankings in Databricks One.
  • Recents and favorites in discovery: Recents and favorites are now surfaced in the For you page.
>
  • Databricks Appsのフィルタリング: Databricks Oneのユーザーは、Databricksアプリを参照する際に、利用可能なアプリのみを表示するか、利用不可能なアプリも含めて表示するかをフィルタリングできるようになりました。利用不可能なアプリとは、停止中、デプロイ中、クラッシュしているなど、実行中以外の状態のアプリのことです。詳細はアプリの状態をご覧ください。
  • ダッシュボードウィジェットのタイトル検索: 検索はダッシュボードウィジェットのタイトルにも対応するようになりました。
  • 検索ランキングの強化: Databricks Oneでは、認証済みコンテンツおよびお気に入りコンテンツが検索ランキングで優先されるようになりました。
  • 最近使ったアイテムとお気に入りのディスカバリー: 最近使ったアイテムとお気に入りがFor youページに表示されるようになりました。
Dashboard subscriptions, custom calculations, and visualization improvements
> ダッシュボードのサブスクリプション、カスタムの計算、可視化の改善
January 22, 2026
  • Account-level user subscriptions: Dashboard authors can now subscribe account-level users to scheduled dashboard updates. See Manage scheduled dashboard updates and subscriptions.
  • Custom calculation references: Custom calculations can now reference custom calculations on dashboard datasets.
  • Cumulative totals on stacked bar charts: Stacked bar charts can now display the cumulative total of values.
  • Improved tooltips on combo and dual axis charts: Tooltips are now easier to access when hovering over combo and dual axis charts.
  • Fixed static widget filter drop-down menu values: Resolved an issue where values in static widget filter drop-down menus were not available when applied on a metric view.
  • Fixed text widget character limit error message: Resolved an issue where trying to save a dashboard with text widgets exceeding 50,000 characters would fail without an error message. An error message now appears informing the author.
>
  • アカウントレベルのユーザーサブスクリプション: ダッシュボードの作成者は、アカウントレベルのユーザーを定期的なダッシュボードの更新にサブスクライブできるようになりました。定期的なダッシュボードの更新とサブスクリプションの管理を参照してください。
  • カスタム計算の参照: カスタム計算では、ダッシュボードデータセット上のカスタム計算を参照できるようになりました。
  • 積み上げ棒グラフの累計合計: 積み上げ棒グラフに、値の累計合計を表示できるようになりました。
  • コンボチャートと両軸チャートのツールチップの改善: コンボチャートと両軸チャートにカーソルを重ねると、ツールチップがアクセスしやすくなりました。
  • 静的ウィジェットのフィルタードロップダウンメニューの値の修正: メトリックビューに適用すると、静的ウィジェットのフィルタードロップダウンメニューの値が利用できないという問題を解決しました。
  • テキストウィジェットの文字数制限エラーメッセージの修正: テキストウィジェットに50,000文字を超える入力をしようとすると、エラーメッセージなしで保存に失敗する問題を解決しました。作成者に通知するエラーメッセージが表示されるようになりました。
Genie lineage and entity matching improvements
> 遺伝子系譜および実体マッチングの改善
January 22, 2026
  • Unity Catalog lineage for Genie spaces: Genie spaces are now discoverable in Unity Catalog lineage.
  • Bulk actions for entity matching: Users can now make bulk actions for entity matching (add, remove, and refresh).
>
  • Genie スペースの Unity Catalog lineage: Genie スペースは Unity Catalog lineage で発見可能になりました。
  • エンティティ マッチングのバルクアクション: ユーザーはエンティティ マッチングにバルクアクション(追加、削除、更新)を行えるようになりました。
Front-end PrivateLink for performance-intensive services (Beta)
> フロントエンド PrivateLinkのパフォーマンス集約型サービス (ベータ版)
January 22, 2026

You can now use AWS front-end PrivateLink for private connectivity to performance-intensive services like Zerobus Ingest and Lakebase Autoscaling. See Configure front-end PrivateLink for performance-intensive services.

>

AWSのフロントエンドPrivateLinkを使用して、ZerobuスインジェストやLakebaseオートスケーリングなどのパフォーマンスの高いサービスに対する秘密の接続を行うことができます。パフォーマンスの高いサービスのフロントエンドPrivateLinkの設定を参照してください。

Lakebase is now generally available
> 湖底（レイクベース）は現在一般に利用可能となりました。
January 22, 2026

Lakebase is now generally available. Lakebase Autoscaling and Lakebase Provisioned are now unified in a single UI. GA includes autoscaling, scale-to-zero, instant branching, automated backups, point-in-time recovery, storage up to 8 TB, and expanded region availability.

See Get started with Lakebase Postgres.

> レイクベースは現在一般公開されています。レイクベースの自動スケーリングとプロビジョンドが単一のUIに統合されました。一般公開には、自動スケーリング、ゼロスケール、即時のブランチ作成、自動バックアップ、特定時点への復元、最大8TBのストレージ、拡大したリージョンの提供が含まれます。 「レイクベースPosgresの使い始め」をご覧ください。

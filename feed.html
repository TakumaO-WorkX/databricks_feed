Databricks Docs Updates

Databricks SQL version 2025.25 is rolling out in Current
> データブリックス SQL バージョン 2025.25が現在展開中です。
August 21, 2025

Databricks SQL version 2025.25 is rolling out to the Current channel from August 20th, 2025 to August 28th, 2025. See features in 2025.25.

>

Databricks SQL バージョン2025.25は2025年8月20日から2025年8月28日の間にCurrentチャンネルに配信されています。機能については2025.25をご覧ください。

Dashboard sorting and formatting improvements
> ダッシュボードの並べ替えと書式設定の改善
August 21, 2025

  • Fixed sorting on temporal categorical axes: Resolved an issue where sorting did not work correctly on temporal categorical axes.

  • Numeric formatting as Bytes: Users can now format numeric values as Bytes.

  • Totals in histogram tooltips: Histograms now display totals in tooltips.

  • Axis gridlines respect custom background colors: Gridlines now automatically update based on custom color palette selections for background color.

>

  • 時間カテゴリカル軸の固定ソート: 時間カテゴリカル軸でソートが正しく機能しない問題を解決しました。

  • バイト形式の数値書式: ユーザーは数値をバイト形式で書式設定できるようになりました。

  • ヒストグラムのツールチップに合計表示: ヒストグラムにツールチップに合計値が表示されるようになりました。

  • カスタムの背景色に合わせた軸グリッドライン: グリッドラインは、背景色のカスタムカラーパレットを自動的に反映するようになりました。

Genie SQL validation and space management
> Genie SQLの検証とスペース管理
August 21, 2025

  • SQL expression validation for join relationships: Added validation logic to help authors write correct SQL join relationships. If a join statement contains errors, the validation logic flags them.

  • Fixed cloning issue for spaces: Resolved a bug where cloned spaces did not correctly copy over the associated tables.

>

  • join関係のためのSQLの式の検証: 正しいSQLのjoin関係を作成するために、検証ロジックを追加しました。joinステートメントにエラーが含まれている場合、検証ロジックがそれらを示します。

  • スペースのクローン作成の問題を修正: クローンされたスペースが関連するテーブルをうまくコピーできない不具合を解決しました。

Serverless compute runtime updated to 17.1
> サーバーレスコンピューティングのランタイムが17.1にアップデートされました。
August 19, 2025

Serverless compute for notebooks, jobs, and Lakeflow Declarative Pipelines now uses an upgraded runtime, which roughly corresponds to Databricks Runtime 17.1. See Serverless compute release notes.

> ノートブック、ジョブ、およびLakeflow宣言的パイプラインのサーバーレスコンピューティングは、Databricks Runtime 17.1に相当するアップグレードされたランタイムを使用するようになりました。「サーバーレスコンピューティングのリリースノート」を参照してください。
Set the run-as user for Lakeflow Declarative Pipelines
> Lakeflow宣言型パイプラインのrun-asユーザーを設定する
August 18, 2025

You can now change the identity that a pipeline uses to run updates and the owner of tables published by the pipeline. This feature allows you to set a service principal as the run-as identity, which is safer and more reliable than using user accounts for automated workloads. Common use cases include recovering pipelines when the original owner was deactivated and deploying pipelines with service principals as a best practice.

For information about setting the run-as user, see Set the run-as user.

> パイプラインが更新を実行し、パイプラインが公開するテーブルの所有者を変更できるようになりました。この機能により、ユーザーアカウントを使用するよりも安全で信頼性の高いサービスプリンシパルをrun-asアイデンティティとして設定できます。一般的な使用例には、元のオーナーが無効になったパイプラインの復旧や、ベストプラクティスとしてサービスプリンシパルでパイプラインをデプロイすることが含まれます。 run-asユーザーの設定について詳しくは、「run-asユーザーの設定」をご覧ください。

Databricks Docs Updates

OAuth token federation is now GA
> OAuth トークン連携が一般公開されました
August 26, 2025

OAuth token federation is now generally available. Token federation enables you to securely access Databricks APIs using tokens from your identity provider (IdP). You can configure token federation policies directly in the Databricks UI, or using the Databricks CLI or REST API.

See Configure a federation policy.

> OAuth トークンフェデレーションは現在一般利用可能になっています。トークンフェデレーションにより、ID プロバイダー (IdP) からのトークンを使用して、Databricks API にセキュアにアクセスできるようになります。Databricks UI、Databricks CLI、または REST API を使用して、トークンフェデレーションポリシーを直接設定できます。 詳細については、フェデレーションポリシーの設定を参照してください。
New table property to control Delta table compression
> 新しいテーブルプロパティーによるデルタテーブルの圧縮の制御
August 26, 2025

You can now explicitly set the compression codec for a Delta table using the delta.parquet.compression.codec table property. This property ensures that all future writes to the table use the chosen codec. See Delta table properties reference.

>

デルタ テーブルの圧縮コーデックを明示的に設定できるようになりました。delta.parquet.compression.codec テーブル プロパティを使用してください。このプロパティにより、テーブルへの今後すべての書き込みで選択されたコーデックが使用されます。デルタ テーブルのプロパティのリファレンスをご覧ください。

Data type mapping update for synced tables
> データ型マッピング更新同期されたテーブルのため
August 25, 2025

For newly created synced tables, TIMESTAMP types in source tables are now mapped to TIMESTAMP WITH TIMEZONE in synced tables. In existing synced tables, TIMESTAMP types continue to map to TIMESTAMP WITHOUT TIMEZONE.

See Data type mapping.

>

新しく作成された同期されたテーブルでは、ソーステーブルのTIMESTAMP型が同期されたテーブルではTIMESTAMP WITH TIMEZONEにマップされます。既存の同期されたテーブルでは、TIMESTAMP型はTIMESTAMP WITHOUT TIMEZONEにマップし続けます。

データ型のマッピングを参照してください。

Automatic liquid clustering is now available for Lakeflow Declarative Pipelines
> 自動リキッドクラスタリングがLakeflowディクレアラティブパイプラインで利用可能になりました。
August 25, 2025

You can now use automatic liquid clustering with Lakeflow Declarative Pipelines. Use automatic liquid clustering with CLUSTER BY AUTO, and Databricks intelligently chooses clustering keys to optimize query performance.

See Automatic liquid clustering, create_streaming_table, table, CREATE MATERIALIZED VIEW (Lakeflow Declarative Pipelines), and CREATE STREAMING TABLE (Lakeflow Declarative Pipelines).

>

Lakeflow Declarative Pipelinesを使用して、自動リキッドクラスタリングを使用できるようになりました。CLUSTER BY AUTOを使用して自動リキッドクラスタリングを使用し、Databricksが照会パフォーマンスを最適化するクラスタリングキーを賢明に選択します。

自動リキッドクラスタリング、create_streaming_table、table、CREATE MATERIALIZED VIEW (Lakeflow Declarative Pipelines)、およびCREATE STREAMING TABLE (Lakeflow Declarative Pipelines)を参照してください。

Budget policy support for database instances and synced tables (Public Preview)
> データベースインスタンスとSync済みテーブルに対する予算ポリシーサポート（パブリックプレビュー）
August 25, 2025

You can now tag a database instance and a synced table with a budget policy to attribute billing usage to specific policies. Additionally, custom tags can be added to a database instance for more granular attribution of compute usage to teams, projects, or cost centers.

See Update a serverless budget policy of a database instance, Create a synced table, and Custom tags.

> データベースインスタンスとデータベーステーブルにバジェットポリシーを設定して、特定のポリシーへの課金使用量を特定することができます。さらに、データベースインスタンスにカスタムタグを追加して、チーム、プロジェクト、コストセンターに対してよりきめ細かい計算使用量の帰属を行うことができます。 「データベースインスタンスのサーバーレスバジェットポリシーの更新」、「同期テーブルの作成」、「カスタムタグ」の詳細については、関連のドキュメントを参照してください。
